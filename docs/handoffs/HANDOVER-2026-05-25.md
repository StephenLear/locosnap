### Phase A facts-layer systemic fix — SHIPPED (`45e1090`)

Instead of writing class-by-class bullets for the next 15-20 high-volume DE/PL/AT classes (the original 4-6h hand-written audit plan), shipped a structural change that covers **every class with a KNOWN_SPECS entry** — ~150-200 classes — in one change.

**The mechanism:** the existing year-only `VERIFIED FACT` injection in `trainFacts.ts` (`This class entered service in ${verifiedYear}`) was extended to pass **every KNOWN_SPECS / Wikidata value** into the facts prompt as a single `VERIFIED FACTS` block. The new system-prompt rule forbids the LLM from contradicting any value (builder, year, max speed, power, weight, fleet count, gauge, fuelType, operator, type) in `summary` / `historicalSignificance` / `funFacts` / `notableEvents`. Precedence: `KNOWN_SPECS override > Wikidata > unset`. Class/operator/type from the identification itself are always included.

**Changes:**
- `backend/src/services/trainSpecs.ts` — exported `SpecsOverride` type. New `lookupKnownSpecs(trainClass)` helper returns the WIKIDATA_CORRECTIONS entry for a class (or undefined). Same `toLowerCase().trim()` normalisation as `applyKnownCorrections` ensures parity. No behaviour change for `getTrainSpecs` callers.
- `backend/src/services/trainFacts.ts` — three coordinated changes:
  - New `buildVerifiedFactsBlock(train, wikidata, known)` helper composing the multi-line block.
  - `buildFactsUserMessage` signature widened from `(train, verifiedYear?, language?)` to `(train, wikidata, known, language?)`.
  - `getTrainFacts` call site calls `lookupKnownSpecs(train.class)` synchronously alongside the existing `getWikidataSpecs` lookup.
  - System prompt: new top-of-rules entry forbidding contradiction of any verified-block value.
- `backend/src/__tests__/services/trainFacts.test.ts` — 2 existing tests updated to match the new block format (year now expressed as `Year introduced: 2020` inside the VERIFIED FACTS block; block always emitted even without Wikidata). 4 new tests: (a) block always carries class/operator/type without Wikidata, (b) full KNOWN_SPECS values surface against real BR 114 entry (LEW Hennigsdorf, 160 km/h, 4,220 kW, 37 units), (c) KNOWN_SPECS wins precedence when Wikidata returns a conflicting value, (d) Wikidata fills fields KNOWN_SPECS does not cover (against Class 390 weight + yearIntroduced).
