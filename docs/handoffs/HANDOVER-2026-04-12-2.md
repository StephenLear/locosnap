## Key Decisions
- **3 lifetime scans, not monthly** -- zero conversions meant the paywall was invisible. 3 lifetime ensures users hit the paywall within their first session.
- **Upsell banner on every results screen** -- previously the paywall was entirely reactive. Now every non-Pro user sees "Grow your collection" after every scan.
- **Backend paywall change held for v1.0.19** -- frontend and backend must ship together. Backend stays at 10/month until the frontend build is live.
- **BR 101 speed stays at 220 km/h** -- this is the certified design speed, not DB's self-imposed operational limit of 200. Multiple commenters confirmed the distinction.
- **EN on DE Instagram is no longer a hard no** -- data doesn't support suppression theory. UK downloads 2:1 over Germany changes the calculus.

## Lessons Learned & Gotchas
- **Frontend and backend paywall changes MUST ship together** -- deploying backend limit changes ahead of the frontend causes confusing 429 errors. A 10-second mental simulation ("what happens to a user on the current frontend?") would have caught this instantly.
- **Always bump iOS version before building if current version is live** -- EAS auto-increments buildNumber but NOT CFBundleShortVersionString. Apple rejects builds with a version that's already been approved.
- **UK downloads are 2:1 over Germany** -- this was not in any previous handover. Challenges the German-first content strategy. UK users are converting better despite having zero dedicated UK train content.
- **EAS build credits at 95%+ for the month** -- the wasted build 39 pushed usage higher. Additional builds will be charged at pay-as-you-go. Be conservative.
- **Zero Pro conversions** -- nobody was seeing the paywall because 10/month was too generous. The combination of reducing to 3 lifetime + adding an upsell banner on every result should fix this.
