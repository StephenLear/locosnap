## Lessons Learned & Gotchas
- **Stale-recording trap:** an ad card baked from an OLD app scan is NOT evidence of current backend state. EN57's "1,700" was already fixed; the recording just predated the fix. Always check the fix date vs the recording date (and read CHANGELOG) before "fixing" from a screenshot.
- **KNOWN_SPECS / WIKIDATA_CORRECTIONS lookup is EXACT-match on the full lowercased class string.** Vision returns `"Newag Dragon 2 (E6ACTadb)"` *with the parenthetical*, so bare keys like `e6actadb` / `dragon 2` silently no-op. New overrides must key the FULL class string the vision layer actually emits — confirm with a fresh scan.
- **Operator is vision-only** — there is no `operator` field in `SpecsOverride`. Operator fixes go in `vision.ts`, and they're heuristic (livery reading), not guaranteed.
- **The Dragon speed key is matched to the LIKELY class string** — a fresh Dragon scan post-deploy is the clean confirmation it lands (silent no-op if the exact string differs).
- **EN57 permission-clean footage = own scan only** (the prior 20.7% EN57 winner used a third-party *snow* lineside clip). Own-scan frame 1 has the reticle/scan-text intrinsically — this is as strong as the hero gets without a permission-clean lineside clip (DM-and-ask would be the real upgrade).
