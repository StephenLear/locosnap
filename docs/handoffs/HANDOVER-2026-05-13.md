## Lessons Learned & Gotchas

- **Working in a git worktree: absolute paths to the project root go to the MAIN working tree, not your worktree.** `/Users/.../locosnap/docs/CHANGELOG.md` is the main checkout; the worktree's copy lives at `/Users/.../locosnap/.claude/worktrees/<name>/docs/CHANGELOG.md`. The Edit tool's absolute-path argument doesn't auto-resolve to the current working directory's worktree ‚Äî it goes wherever the path says. Always use relative paths or worktree-prefixed absolute paths when editing project files from a worktree session. Wasted ~5 min today reverting + reapplying.
- **Before writing customer-facing copy that mentions an offer (trial, discount, free X), VERIFY the offer is actually configured on the relevant platform.** Read `project_revenuecat_topology.md` first. Apple intro-offers and Play offers each have different product types ("Free trial" vs "Pay Up Front" vs "Single payment phase" vs "Pay as you go") with different surfacing rules. The infra-side state can drift from the product-side intention.
- **RC's "Cancelled (still in active period)" status is the default display for any prepaid Play subscription with no scheduled renewal.** It is NOT evidence of an active cancellation event. The Play prepaid base plan model has no auto-renew by design. Forward rule: when reading RC churn data, segment by product-type (prepaid vs autorenew). Any analysis that mixes the two is structurally ambiguous. Pre-2026-05-10 Play customers were all forced onto prepaid because the autorenew variants didn't exist yet ‚Äî the first true churn-baseline cohort is the post-2026-05-10 `*-autorenew` buyers.
- **Industry-data lifts compound non-linearly.** Adapty / RevenueCat case-study lifts assume baseline = cold-hard-paywall. A LocoSnap soft-scan-budget paywall hit at scan 6 is already a warmer setup than the benchmark baseline, so the marginal lift from adding a trial is smaller than the case-study number. Always discount cited lifts by the "what's your actual baseline" question.
- **Hybrid utility+collection apps face a real strategic fork.** Utility-only data says hard-paywall wins (Adapty median 12.11% vs freemium 2.18%); habit/collection data says free-core wins (Letterboxd / Strava / iNaturalist Pattern A). LocoSnap is both ‚Äî and the right model depends on which the next 90 days prioritises (short-term conversion vs 12-month moat). Don't ship interventions that push opposite directions at the same release.
- **TikTok comment reply rules per `feedback_tiktok_reply_length`** ‚Äî 150-char hard cap; per-platform limits differ; DMs have higher limits. Public replies should be short, neutral, and never engage with a negative public commenter directly when the negative comment is dissuading other prospects. Private DM is the better intel-gathering channel.
- **Polish ad-copy pattern (carried from yesterday):** family/collection framing ("Dragon 1 vs Dragon 2", "z tej samej rodziny") beats generational/successor framing for Polish rail fans. Today's PL reply to the screenshot-poster used "Dziƒôki za scany do tej pory" in the same spirit.
