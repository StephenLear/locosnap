## Lessons Learned & Gotchas
- **A hallucinated class STRING (e.g. "ST22") is the worst misID case** — it bypasses every per-class override. Fix at the canonicalisation layer (`classNames.ts EXPLICIT_ALIASES`), not per-layer.
- **A misID rule can silently fail on a NEW angle.** The Loram rule worked for a side/night shot but not a front-3/4 daytime shot — different cues in frame.
- **Hostinger cancellation left a latent stale ALIAS** that downed the site weeks later — "zero impact" was wrong. Future Hostinger-cancellation cleanup: check for leftover `connect.hostinger.com` ALIAS/records.
- **Supabase RLS (migration 018) locks raw spot lat/lng to the owner** — any cross-user location feature MUST be a SECURITY DEFINER aggregate with k-anonymity, never raw coords/user_id.
- **Monaco (Supabase SQL editor) overtype** handles typed parens/quotes fine; `||` on numerics needs `::text` casts.
