## Lessons Learned & Gotchas

- **Vision-rule positioning matters more than content** when the model has no training-data anchor for the class. Class 69 v1 and Sm v1 both had complete content but landed deep in the prompt; model finalised its answer before reaching them. Pre-flight-check positioning (same pattern as Mireo / BR 151 / Taigatrommel) is mandatory for classes the model doesn't natively know.
- **The "negative guard" half of a disambiguation rule tells the model what NOT to answer, but doesn't help it find the RIGHT answer.** Class 69 v1 warned against Class 37/56/60/66 but the model just picked Class 33 (not in the forbid list). Future rules need the forbid-list to cover the full category, not just the one obvious wrong answer.
- **When an agent runs research based on a prompt, the prompt's gaps become its gaps.** Paywall research agent assumed monthly-only pricing because my prompt quoted "£2.99–£4.99/mo" without mentioning annual. Agents hallucinate less when the prompt is complete; when it isn't, they extrapolate, and the extrapolation can be wrong. **Always include ALL known facts about the subject in the prompt, even ones that feel redundant.**
- **Tester-evangelist relationships compound fast.** In one session Steph progressed from "blocked on install" → "UX research respondent" → "confirmed streamer tier" → "source of the most important paywall feedback of the day". The relationship compounds when we treat each interaction as collaborative. Pattern logged to `memory/project_tester_evangelist_pattern.md`.
- **Cache-serves-stale-answer is not actually a risk in vision.ts failures.** I suspected the Class 69 wrong-answer-post-deploy could be cache. Confirmed it's not — `trainCache.ts` keys on the AI's returned `{class, operator, language}`, so a different wrong answer post-fix would hash to a different key anyway. The issue is always the vision rule, never the cache.
- **Supabase keys are NOT in local `backend/.env`** — verified 2026-04-24. Only `RESEND_API_KEY` is local. All other prod keys (Supabase, PostHog, Sentry, RevenueCat) are Render-only. `credentials_locations.md` memory updated with this clarification.
- **`docs/handoffs/`, `docs/plans/`, `docs/design/` files are untracked in git.** Not build-critical, but session artefacts exist only on disk. Worth a `git add docs/ && git commit` whenever someone has a free moment — 12+ handovers and plans accumulated across recent sessions. Not blocking anything.
