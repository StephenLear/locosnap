### Finnish tester crash fix (v1.0.12)
- Analysed Samsung bug report (DOC-20260403-WA0029.zip) — confirmed `data_app_crash` (Java managed exception), crash loop every 10-20s, artd log showing interpreted execution mode (`filter 'verify' executable 'false'`). Full stack trace not accessible in Samsung DOC-WA format.
- Root cause identified: `import "../i18n"` in `_layout.tsx` ran `i18n.use(initReactI18next).init()` as a module-level side effect during JS bundle evaluation — before any component mounted or native bridge was ready. On Samsung S24 Android 16 in interpreted mode, this caused a crash loop.

## Lessons Learned & Gotchas

- **Android EAS builds: always `--profile preview`** unless explicitly asked for Play Store. Production = AAB (useless for direct distribution). Preview = APK.
- **Samsung DOC-WA bug reports don't include stack traces** — the dropbox entries are truncated summaries. If you need the actual exception, ask for `adb logcat` or a full Android bug report via Developer Options.
- **`initImmediate: false` is required** when calling `i18n.changeLanguage()` immediately after `initI18n()` in the same synchronous flow — without it, init is async and changeLanguage runs on an uninitialised instance.
- **settingsStore.initialize() calls i18n.changeLanguage()** — the order `initI18n()` → `initializeSettings()` in the same useEffect is load-bearing. Do not separate them.
- **Polish TikTok comments are a quality signal** — the EU07 commenter produced two actionable technical corrections. Engaging with these comments is worth doing promptly.
- **Always read memory files before editing** — mid-session edits change the file content; targeting strings from earlier in the session can corrupt entries.
