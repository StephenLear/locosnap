### 3. Shareable card feature — implemented, not yet on a working build

All code is on `main`. 39/39 tests passing.

**What was built:**
- Hidden static `shareCardRef` View: `position: absolute, top: 0, left: 0, transform: [{ translateX: -9999 }]`, fixed 400x580px, `collapsable={false}`. This is the capture target for `captureRef`.
- Fixed `handleShare`: captures `shareCardRef` (not the animated `cardRef`), no emoji, share text is `"Guess what I just spotted and added to my collection near [CITY]. Identified with LocoSnap."` (city from reverse geocoding, silently omitted if unavailable).
- New `handleSave`: `MediaLibrary.requestPermissionsAsync()` -> `captureRef(shareCardRef)` -> `saveToLibraryAsync()`.
- Reverse geocoding on mount: `Location.reverseGeocodeAsync({ latitude, longitude })` with 2s timeout.
- Three-button action row: [Save] [Share] [Details]. Loading states (hourglass icon, 0.5 opacity).
- `app.json`: added `NSPhotoLibraryAddUsageDescription` and `expo-media-library` plugin (required for App Store submission).
- Version bumped to `1.0.7` (1.0.6 was already approved/live on App Store).

**The critical bug fix (last commit of session):**
The original implementation used `left: -9999` on the hidden card. iOS GPU skips rendering views that far outside layout bounds — `captureRef` captured nothing, catch block swallowed it silently, buttons appeared dead. Fixed to `transform: [{ translateX: -9999 }]` which keeps layout position at (0,0) so iOS renders the view, but moves the visual output off-screen.

## What Needs to Happen Next Session

### Step 1: Pre-build code review (mandatory — do not skip)

Before triggering any EAS build, read `frontend/app/card-reveal.tsx` in full and verify:

1. `shareCard` style uses `transform: [{ translateX: -9999 }]` (not `left: -9999`) — this is the captureRef fix
2. `shareCardRef` is referenced on the correct `<View>` (line ~578), has `collapsable={false}`
3. `handleShare` and `handleSave` both target `shareCardRef`, not `cardRef`
4. `app.json` has `NSPhotoLibraryAddUsageDescription` in `infoPlist` and `expo-media-library` in `plugins`
5. Version in `app.json` is `1.0.7`
6. Run `npx tsc --noEmit` — only acceptable error is the pre-existing `_layout.tsx TS2459` unrelated to this feature
7. Run `npm test` — all 39 must pass

### Step 2: Build and submit

```bash
cd frontend
eas build --platform ios --profile production --non-interactive
eas submit --platform ios --latest --non-interactive
```

### Step 3: TestFlight test checklist

Once the build appears in TestFlight (version 1.0.7, build 33+):
- Scan a train

- Share text must match: `"Guess what I just spotted and added to my collection near [CITY]. Identified with LocoSnap."` (with city if GPS available)
- No emoji anywhere in share text
