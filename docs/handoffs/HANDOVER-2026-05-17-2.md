## Lessons Learned & Gotchas

- **Apple tier ≠ raw FX. Play tier ≈ raw FX.** Apple's tier system has purchasing-power adjustment baked in (DE monthly auto-tier was €2.99, PL annual auto-tier was 99.99 zł not the 150 zł raw FX would give). Play's auto-conversion is much closer to raw FX (PL annual was 149.99 zł, CZ 849.99 Kč). Implication: Play needs bigger per-country cuts than Apple. Always spot-check before recommending.
- **Memory files decay fastest for git/build state.** A 2-day-old memory file claimed v1.0.31 was "READY TO BUILD" but the changelog showed it had shipped. The CLAUDE.md rule "Before asserting what is or is not in a build — always read `docs/CHANGELOG.md` first" exists for exactly this reason. When a system-reminder flags a memory as stale, treat that as a hard signal — re-verify against the changelog.
- **expo-localization remains a permanent landmine.** It was uninstalled in v1.0.11 because the native module crashed Android-16 + Finnish-locale devices at startup before Sentry could initialise — invisible to monitoring. If device-locale detection is ever revisited, the only safe path is `Intl.DateTimeFormat().resolvedOptions().locale` (already proven safe in `deriveInitialCountry`). Permanent guard: `feedback_no_expo_localization.md`.
- **`eas submit` non-interactive flag interactions.** `--non-interactive` requires `--id <build-id>` (or `--latest`). Don't fire without it.
- **When walking through a multi-step GUI process and the user gets lost, drop to one click per turn.** Batched step lists ("do these 7 things") fail; literal single-click instructions ("click the blue + next to Subscription Prices") work.
- **Back-translation is the right verification pattern for non-speaker copy approval.** Cheap, fast, catches howlers, lets the user sign off without external review. Same pattern works for both in-app strings and store metadata.
- **"öööhhh" + a tag in a German trainspotter comment is creator-to-creator banter, not praise.** Safer responses than "thanks very much" are: heart the comment, tag the named creator with a checkmark, or no reply. Saved for next time.
- **Frontend version in `app.json` is independent from EAS auto-increment versionCode.** EAS handles versionCode bumps automatically (production profile has `autoIncrement: true`); we only had to bump `version` from 1.0.31 to 1.0.32 manually.
