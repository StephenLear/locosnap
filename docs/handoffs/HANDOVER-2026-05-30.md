## Key Decisions

- **Project scope, not user scope.** User chose project scope so the tool is tied to the LocoSnap repo, not global.

## Lessons Learned & Gotchas
- **`claude mcp add` is a key-leak risk.** It embeds secrets in plaintext in `.mcp.json` and leaves the file un-ignored. Always gitignore `.mcp.json` (now done) and verify with `git check-ignore .mcp.json` after any MCP install.
- **MCP tools don't load mid-session.** `mcp__resend__*` will only appear after a Claude Code restart / new session in this project. Don't try to call them in the session that installed them.
- **The Resend MCP does NOT expose a suppression-list tool.** Post-batch checks are done by reading `last_event` per email via `list emails` / `get email` — same field as the dashboard CSV, different route. Un-suppressing still requires the dashboard.
- **Don't trust handover checklist prose about git state — verify.** The 2026-05-29 note said the handover was "uncommitted"; `git log origin/main..HEAD` showed it was already pushed. (Reinforces the existing CLAUDE.md rule.)
