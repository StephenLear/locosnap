## Lessons Learned & Gotchas

- **RC dashboard product attachment is per-store, per-package, and silent on the Android side until you check.** Just because the iOS App Store products were attached to a package does NOT mean Android will work. The Play Store product dropdown defaults to "No product" until explicitly attached. Without this, `Purchases.getOfferings()` on Android would return packages with no Play product, and the paywall would silently render empty for Android users. **No error, no warning, just no products.** This is the #1 thing to check on any iOS-first launch when Android goes live.
- **RC's "production purchases use this entitlement" warning is generic and fires on any detach from a live entitlement, regardless of whether the specific product being detached has any real-world purchases.** Test Store placeholder products produce the same warning as real Play Store products. Read the dialog literally, but the actual blast radius is determined by which product is being detached, not by the warning text.
- **The MCP Chrome extension only sees tabs in its own tab group.** Existing tabs in other windows are invisible. Either drag the target tab into the MCP window, or have the extension navigate fresh (which requires re-login for any authenticated dashboard). For dashboards with 2FA, dragging is far less friction.
- **The Pokémon Go comparison is now the canonical word-of-mouth framing for LocoSnap.** Logged in `project_tester_feedback.md`. This is the kind of phrase organic users reach for to describe the app to a stranger — reuse in store descriptions, ad copy, Reddit pitches. The fact that it was used unprompted by a beta tester writing in German validates it as cross-cultural shorthand.
- **`react-native-purchases` v9.9.0 is on the codebase.** Useful to know when RC dashboard fires a "supported only by Android SDK v6+ or equivalent" warning — it's harmless because v9.9.0 is well above v6. If a future session is reading this and the warning fires again, this is the answer.
- **Always check `docs/handoffs/` newest-first when the user references prior-session state.** Saves a round-trip versus guessing or asking the user to recap.
