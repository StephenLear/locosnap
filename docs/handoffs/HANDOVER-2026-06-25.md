## Lessons Learned & Gotchas
- **When a third-party API call suddenly fails universally, check whether the MODEL/endpoint was deprecated BEFORE checking billing/quota.** dall-e-3 retirement is a known 2026 event. Model-first would have reached the answer in one step. (Generalises [[feedback_play_billing_diagnosis]] top-down rule to model deprecation.)
- **`gpt-image-1` is NOT a drop-in for `dall-e-3`:** different `size` enum (no `1024x1792` — use `1024x1536`), different `quality` enum (no `hd` — use low/medium/high), **no `style` param**, and it returns **base64 (`b64_json`), never a URL.** Each mismatch is its own 400. Verify against current OpenAI docs before coding.
- **Blueprint failures are currently swallowed** — no Sentry, generic error stored. Any future provider issue will again be invisible until a user reports it. Fix the observability as part of the migration.
- **Frame-1 MOTION gate holds for feature-reveal ads too** — a static/head-on hero (ICE 1, like BR 111) bails at 0:01 even with strong downstream payload. High like/save among stayers ≠ rescued reach.
- **`gpt-image-1` confirmed cost-relevant:** quality tier directly drives per-blueprint cost ($0.01 mini → $0.25 high). Confirm the tier before shipping.
