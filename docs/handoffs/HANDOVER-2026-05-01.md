## Lessons Learned & Gotchas
- **Memory about deploy state can be stale.** Earlier in session I committed BR 110 backend changes on the `main` branch directly (cleaner than waiting for v1.0.23 merge since these are loco-data fixes, not feature work). User confirmed this is the right pattern — backend fixes that block ad work go to `main` immediately.
- **The first session-recording is often pre-deploy.** Always sanity-check whether the recording was made BEFORE or AFTER the latest backend commit lands on Render. Render auto-deploy from main typically takes 3-5 min.
- **Class number collisions are still the #1 rarity-hallucination source.** BR 110 was getting BR 155 specs (5,400 kW / 120 km/h / freight) + BR 151/155 successor narrative. The number-similarity in "1XX" classes consistently confuses the rarity layer. Same pattern as BR 151 vs ČD 151, BR 232 vs Siemens, BR 412 vs BR 408.
- **`scale=1080:1920,fps=30` works without `force_original_aspect_ratio`** when source is already same aspect — saves arguing with ffmpeg over flag compatibility on macOS Homebrew builds.
- **Yellow Impact bold + thick black outline at MarginV 260 from bottom** = the "biggest impact" subtitle style for these short-form ads going forward. Saved in `/tmp/br110ad/subs.ass` for reference/reuse.
