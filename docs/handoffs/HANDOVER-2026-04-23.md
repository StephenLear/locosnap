## What Worked

- **Submit-as-draft flow** — `releaseStatus: draft` in `eas.json` gave us a gate between the EAS upload and the actual release action. That gate saved the session when the permissions quick-check rejected versionCode 9 — we could just rebuild and re-submit without fighting a half-live release.
- **100% rollout decision** — no staged throttle, fully global. Right call for a zero-install production base. Would have been wrong for an app with real existing users.
- **Permissions blockList pattern** — manual permissions + `blockedPermissions` together is the clean fix for unwanted auto-added permissions from Expo plugins. Keeps Photo Picker API flow working without touching the native layer.
- **Managed publishing OFF** — auto-publish on approval means the moment Google says yes, it's live. No second action from us. For a solo-dev launch this is the right default.
- **Tester review pre-warm** — asking for a Play Store review in the thank-you email, *before* the app is live, promising a follow-up with direct link on approval day. Creates a pre-baked audience for day-one reviews rather than scrambling to ask after launch.
- **Resend Python script pattern** — one-shot `/tmp/send_thanks.py` with inline recipient list, HTML conversion from plain text, mandatory headers. Cleaner than maintaining a long-lived mailer. Deleted after use.

## Lessons Learned & Gotchas

- Google Play 2024 permissions policy is strict on `READ_MEDIA_IMAGES` / `READ_MEDIA_VIDEO`. Any app with only infrequent photo access must use the Photo Picker API and cannot hold those permissions. This is a silent landmine for any Expo/React Native app that default-imports `expo-image-picker` and `expo-media-library`. Keep `blockedPermissions` on by default going forward.
- Play Console store-listing language list is the gate for release-note language tabs. You can't add a de-DE release note if de-DE doesn't exist as a store listing. Plan the store-listing translation first, release notes second.
- Release notes do NOT carry over when a draft release is replaced by a new AAB submit. Budget 2 minutes to re-paste EN + DE.
- EAS build credit exhausted for April — on pay-as-you-go for the rest of the month. Don't queue builds casually.
- The `/tmp/send_thanks.py` pattern for bulk Resend works cleanly: load key from env, HTML-wrap plaintext with logo-at-top, rate-limit via `time.sleep(0.4)`. Delete after use so no secrets in shell history.
- Handover files in `docs/handoffs/` have been accumulating untracked (10+ of them). Not build-critical — text docs — but probably worth a housekeeping commit soon. Logged as a non-blocker.
