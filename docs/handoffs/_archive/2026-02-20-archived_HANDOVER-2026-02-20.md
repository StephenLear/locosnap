## What Worked
- `eas submit --platform ios --latest` — picks up most recent finished build automatically, no need to specify build ID
- `eas build:list --platform ios --limit 1` — quick way to check latest build status and get build URL
- Running `eas build` without `| head -N` pipe — previous session accidentally truncated output hiding the build URL

## What Didn't Work (and Fixes)
- **First `eas build` attempt (build 18) had output truncated by `| head -30`**: The build URL and progress were hidden. Process was killed and restarted without the pipe. Build number auto-incremented to 19 as a result.
- **`eas submit` ran twice, second attempt failed with "already submitted"**: Not a real error — the build was already received by Apple from the first attempt. Exit code 1 but submission was successful.

## Key Decisions
- **Removed guest guard entirely from paywall** rather than adding a "sign in to purchase" prompt: Apple guideline 5.1.1 requires IAP to be accessible without mandatory registration. The purchase functions (`purchasePro`, `purchaseBlueprintCredits`) already work with RevenueCat anonymous users — no Supabase account needed for the transaction. `syncProStatus` is already guarded by `if (user)` so nothing breaks for guests.
- **Location string written specifically**: Apple rejected the previous submission for a vague location purpose string. New string: *"LocoSnap uses your location to record where you spotted each train, so you can see your sightings on a map."* — directly explains the use case.

## Lessons Learned & Gotchas
- **Never pipe `eas build` output through `head -N`** — you'll lose the build URL and progress indicator
- **Build number auto-increments on every `eas build` call**, even if the previous one was killed partway through. Check `eas build:list` after a failed/killed build to know the current build number before starting another
- **`eas submit` exit code 1 "already submitted" is not a real failure** — it means Apple already received it, which is success
- **Apple reviewer note is critical**: Add notes in App Store Connect → App Review Information → Notes before the reviewer picks it up. Explain guest mode as workaround for OTP email issues
- **Build 19 is the correct build despite build number gap** — build 18 was never actually compiled, just had its number reserved
