## Key Decisions
- **Beta Pro via env var, not Supabase direct-write**: Supabase `is_pro` gets overwritten by `syncProStatus()` on every app launch, so setting it directly in the DB doesn't persist. The env var approach intercepts the sync itself.
- **Compression on native only, not web**: `expo-image-manipulator` is a native module; web uploads use `fetch()` + `File` objects directly and already handle reasonable sizes from browser file pickers.
- **No RevenueCat promotional entitlements for beta**: Would require testers to have existing RC records. The code override is simpler and doesn't depend on external service state.

---

## Lessons Learned & Gotchas
- **`syncProStatus()` overwrites Supabase on every launch** — setting `is_pro: true` in Supabase directly will be undone next time the user opens the app. Any Pro grant must go through the `syncProStatus` path or be applied before it runs.
- **RevenueCat App User ID = Supabase UUID** — RC searches by App User ID (UUID), not email. Searching RC dashboard by email finds nothing.
- **EAS preview profile must include ALL `EXPO_PUBLIC_*` vars** — they are not inherited from `production`. Each profile is isolated. If a key is missing from `preview`, the feature silently doesn't initialise.
- **`SENTRY_DISABLE_AUTO_UPLOAD=true` required in any profile without a Sentry auth token** — otherwise EAS build fails at the Sentry upload step.
