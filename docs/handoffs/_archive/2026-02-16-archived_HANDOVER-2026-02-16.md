## What Worked

- The SCANNER colour constant pattern (local to each screen) keeps scanner-specific values co-located

## What Didn't Work (and Fixes)
- **Guest sign-in navigation loop**: `AuthGate` in `_layout.tsx` treated guests as "authenticated" (`isAuthenticated = session !== null || isGuest`) so tapping "Sign in to save" would immediately bounce back to home. Fixed by changing to `isFullyAuthenticated = session !== null` and only blocking sign-in access for real sessions, not guests.
- **Auth state listener resetting guest flag**: `onAuthStateChange` in `authStore.ts` had `isGuest: !session` which would reset guest state whenever Supabase fired an event. Fixed by preserving `wasGuest` state: `isGuest: session ? false : wasGuest`. Same fix applied to `setSession`.
- **Magic link "Send Failed"**: The `.env` file is missing `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_ANON_KEY`. The code falls back to placeholder values (`https://your-project.supabase.co`). User needs to set up Supabase credentials — this is the current blocker.

## Key Decisions
- **Teal `#00D4AA` as global accent**: Chose to update the theme-level `colors.accent` rather than keeping orange on other screens. This ensures visual coherence across all screens and reduces maintenance.
- **Blue `#0066FF` as primary**: Used for secondary actions (upgrade button, magic link send button) to differentiate from the teal accent.
- **Preserved SCANNER local constants**: The home screen and paywall keep their own `SCANNER` object with teal/blue variants for screen-specific styling (glows, subtle backgrounds) while still referencing `colors.accent` for shared elements.
- **`upgradeBtn` uses `colors.primary` (blue)**: Differentiated from the teal accent to make the Pro upgrade button stand out as a distinct CTA.

## Lessons Learned & Gotchas
- **AuthGate guest logic is tricky**: Any code that checks `isAuthenticated` must distinguish between "has a real Supabase session" and "is using guest mode". The `||` combining both was the root cause of the navigation loop.
- **Supabase `onAuthStateChange` fires unexpectedly**: It can trigger on app state changes, token refreshes, etc. Any state mutation in this callback must be careful not to clobber guest state.
- **Hardcoded rgba values don't follow theme**: While `colors.accent` propagates automatically, any hardcoded `rgba(255, 107, 0, ...)` must be found and replaced manually. Use `Grep` for `ff6b00|ff8533|255, 107, 0` to catch them all.
- **File write tool requires a recent read**: If you read a file much earlier in the conversation, you may need to re-read it before writing.
