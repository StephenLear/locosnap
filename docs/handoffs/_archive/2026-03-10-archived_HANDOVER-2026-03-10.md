## What Was Done

## What Worked
- Xcode Organizer crash reports: the `NO_CRASH_STACK` entry gave a readable stack trace pointing to `ObjCTurboModule::performVoidMethodInvocation` — a TurboModule native module call failing
- EAS build + submit pipeline worked cleanly once build ID was passed explicitly to `eas submit --id`
- `autoIncrement: true` in eas.json handled build number bumping automatically (19 → 20 → 21)

## What Didn't Work (and Fixes)
- `eas submit --non-interactive` without `--id` failed with "must specify archive source" → fixed by passing `--id <build-id>` explicitly
- First submit attempt (build 20) failed: "already submitted this version" because v1.0.0 was already in ASC → bumped version to 1.0.1 in app.json and rebuilt

## Key Decisions
- **v1.0.1 as patch version**: Correct — this is a pure crash fix, no new features
- **`expo-notifications` added to plugins**: Missing from app.json plugins array, which meant the native notification TurboModule wasn't properly registered in the iOS build
- **`setNotificationHandler` moved inside function**: Was running at module import time (top-level of notifications.ts), before React Native's TurboModule system was initialised on RN 0.81.5
- **`scheduleStreakReminder` gated on permission**: Was called unconditionally on every first launch; now only fires if `registerForPushNotifications` returns a token (i.e. user granted permission)

## Lessons Learned & Gotchas
- RN 0.81.5 uses new architecture (TurboModules) by default — calling native module methods at module import time (outside React lifecycle) will crash on first launch
- `expo-notifications` must be in the `plugins` array in app.json, not just in dependencies — without it the native module isn't linked properly in the iOS build
- EAS `autoIncrement` only bumps the build number (CFBundleVersion), not the version string (CFBundleShortVersionString) — you must manually bump `expo.version` in app.json when resubmitting a new version
- `eas submit --non-interactive` requires `--id <build-id>` to specify which build to submit
