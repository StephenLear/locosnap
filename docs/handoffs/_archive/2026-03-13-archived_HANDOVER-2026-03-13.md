## What Worked
- Pure screen recordings (no device frames, hands, or external footage) resolved the Guideline 2.3.4 rejection
- FFmpeg re-encoding pipeline: trim → scale to 886×1920 → enforce 30fps → H.264/AAC output
- PIL image resize for screenshots: `Image.resize((1290, 2796), Image.LANCZOS)`
- Uploading blueprint screenshot as Screenshot 1 so the result screen (not camera) is shown in App Store search

## What Didn't Work (and Fixes)
- **Apple rejected preview (Guideline 2.3.4)** — previous preview contained TikTok-style content (real footage, device frame, hands) → Fixed: replaced with clean screen recordings
- **Wrong dimensions (1206×2622)** — Apple requires exactly 886×1920 for 6.9" previews → Fixed: FFmpeg `scale=886:1920`
- **Frame rate too high (60fps)** — Apple requires max 30fps → Fixed: FFmpeg `fps=30` filter
- **Preview 2 showed blueprint spinner** — looked unfinished, user rejected it → Abandoned, only Preview 4 used
- **Preview 3 was 13 seconds** — under Apple's 15-second minimum → Not used

## Key Decisions
- **Preview 4 only:** User reviewed all three recordings; Previews 2 and 3 were rejected by user. Only Preview 4 (27s to end of recording) made the cut.

## Lessons Learned & Gotchas
- Apple's "uploading and reviewing" state for previews can take 10–20 minutes — normal, just wait
- App Store German localisation is a **separate** submission step from the English listing — can only be done once current review clears
- The keyword field is exactly 100 characters — prepared strings fit exactly, do not add trailing spaces or commas
- Screenshot 1 position matters enormously — it's shown in App Store search results before users tap through
- App Store previews: 886×1920px, H.264, max 30fps, 15–30 seconds — all four must be correct or it rejects
- **Uncommitted frontend changes exist** — `frontend/app.json`, `frontend/app/_layout.tsx`, `frontend/services/notifications.ts` are modified but not committed. Not related to this session but should be committed before next EAS build.

### FFmpeg Commands (for reference if more previews needed)
```bash
# Trim + resize to 886x1920
ffmpeg -i input.mp4 -ss 00:00:27 -vf "scale=886:1920" -c:v libx264 -crf 18 -pix_fmt yuv420p -c:a aac -b:a 192k output-final.mp4

# Convert to 30fps
ffmpeg -i output-final.mp4 -vf "fps=30,scale=886:1920" -c:v libx264 -crf 18 -pix_fmt yuv420p -c:a aac -b:a 192k output-30fps.mp4
```
