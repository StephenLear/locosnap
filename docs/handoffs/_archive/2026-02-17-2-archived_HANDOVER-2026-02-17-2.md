## What Didn't Work (and Fixes)
- **ScrollView-only approach (builds 10-11)**: Using `ScrollView` for blueprint display was fundamentally limited ‚Äî vertical-only in portrait, and fitting to height in landscape just shrank the image. User feedback: "the scroll doesnt work left to right" and "landscale just reduces the size of the picture" ‚Üí **Fix**: Replaced with gesture-based zoomable/pannable view
- **Build 11 landscape sizing**: `height: height - insets.top - 70` with `width: height * (9/16)` made the image too small in landscape since it was constrained to height. The user wanted to actually see detail, not just a smaller version ‚Üí **Fix**: Build 12 fits image to available space and lets user pinch-to-zoom for detail

## Key Decisions
- **Pinch-to-zoom over ScrollView**: ScrollView can't do bidirectional scroll + zoom natively in RN. Using gesture handler + reanimated gives a proper photo-viewer experience (like iOS Photos app)
- **Image sizing strategy**: Calculate available space (screen minus safe area minus bottom bars ~110px), then fit 9:16 image to whichever dimension is tighter. This works correctly in both orientations
- **Zoom limits**: Min 1x (can't zoom out), max 5x. Double-tap toggles between 1x and 2.5x. Snaps back to 1x if barely zoomed (<1.05x)
- **Reset on orientation change**: Zoom and pan reset when `width`/`height` change (device rotation) so the image always starts clean

## Lessons Learned & Gotchas
- `GestureHandlerRootView` MUST wrap the entire app for gestures to work ‚Äî added to `_layout.tsx` around `ErrorBoundary`
- React Native `ScrollView` does NOT support simultaneous horizontal + vertical scrolling ‚Äî it's one direction at a time unless you nest them, which is janky
- The 9:16 blueprint aspect ratio is fundamentally challenging on phone screens ‚Äî in portrait the image is taller than the screen, in landscape it's narrower. Zoom is the right UX pattern for this
- Build credits are at 100% usage for the month ‚Äî additional builds are charged at pay-as-you-go rates
- `useWindowDimensions()` reactively updates on orientation change, which triggers the zoom reset `useEffect`
