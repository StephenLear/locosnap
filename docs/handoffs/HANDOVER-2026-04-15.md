## Lessons Learned & Gotchas

- **10 characters is the actual safe max at 110px Arial Black, not 9.** Prior handover guidance was one character too conservative. Confirmed tested on this build: "DIE BR 143" (10), "646 GEBAUT" (10), "STIRBT AUS" (10) all fit comfortably. "DAS DR-ERBE" (11) is tight but OK. "VERSCHWINDET" (12) and "IS VANISHING" (12) overflow. Plan shorter alternatives in advance for long German words.
- **Always verify the source clip shows the class you're naming.** The Weinrot Dresden Hbf clip OPENS on the Doppelstock Steuerwagen (the non-loco driving end of a push-pull set) — if I'd used the first 2 seconds as a hero beat, German rail fans would instantly correct it in comments (same failure mode as BR 111 "NUR NOCH 4"). Always scrub through provided clips and confirm WHERE the loco actually appears before slicing.
- **"Burned-in creator text" traps on third-party footage.** The Großköris clip has "Groß Köris" text burned into the first ~10 seconds — the original creator's watermark. Can't remove it without heavy processing. Solution: just skip past it and start the beat at 20s. Always scan the opening seconds of any provided clip for burned-in text before picking a start timestamp.
- **Berlin S-Bahn is 750V DC third rail — no pantographs anywhere.** Any disambiguation rule for BR 480 / 481 / 485 that relies on "pantograph visible or not" is wrong by construction. This was baked into the old broken rule and I had to completely rewrite it. File this for future rules involving third-rail-electrified systems (Berlin S-Bahn, Hamburg S-Bahn sections, London Underground, UK Southern Region, DSB S-tog, etc.).
- **The "Blue Tiger" name is exclusively the ADtranz DE-AC33C, NOT the Vossloh Euro 4000.** The old rule conflated the two. This is a factual error in basic rail enthusiast terminology that German commenters would have spotted immediately. When writing ANY rule about a named locomotive, verify the name-to-class mapping against a trusted source (Wikipedia / German rail Wiki) before committing.
- **Check Supabase BEFORE drafting tester status responses.** 90 seconds via browser automation saves writing a dual-scenario draft that turns into throwaway work. Rule now codified.
- **Don't guess pronouns from name "sound".** Finnish, Sámi, Polish, Czech, Japanese, Chinese, Korean, and many other cultures produce names that don't map to gender the way English names do. "Oula" is traditionally male in Finnish/Sámi but defaulting to they/them is still the safer call. Rule now codified.
- **The vision.ts file has at least three factual errors I discovered TODAY alone.** The Vossloh/Blue Tiger mixup, the BR 485 "unpowered trailer" error, and the Berlin S-Bahn pantograph reference. These all existed in the prompt for an unknown length of time, likely since early in the project. There may be more lurking. Consider a future session dedicated to auditing existing rules against authoritative sources, not just adding new ones.
- **Render deploys can trigger Sentry regressions even on backend-only changes.** Saw a REACT-NATIVE-1 error ("Could not connect to LocoSnap servers") regression timed to the deploy. Render spins down briefly on deploy, causing a brief window where the backend is unreachable. Not a bug, not a regression — just the natural deploy blip. Should show up as a temporary spike then self-clear. If it doesn't clear, then investigate.
- **Poland on iOS is a real conversion market now.** 15 Polish downloads in 30 days (0 in prior period) falsifies the previous "Polish content = followers only" rule for iOS. The Android beta data was right at the time but the rule shouldn't have been written as a blanket statement. On iOS, Polish converts. Poland is #3 iOS market behind Germany and the UK.
