## Lessons Learned & Gotchas

- **Two-axis charts are the danger zone.** Cache hit rate (right axis green spike) reads identically to load (left axis blue spike) at first glance. **Fix:** state axis + line + unit explicitly before quoting any number. The skill file has the canonical example.
- **One data point is never a trend.** "Apr 28 $36" is a spike. "Sustained $30-40/day" requires at least 3 consecutive days at similar levels. Memory files persist; premature trend-framing pollutes future sessions.
- **Vercel project ≠ local folder name.** `locosnap.app` is aliased to project `website` (source `~/Projects/locosnap/website/`), NOT `locosnap-landing`. Always run `vercel alias ls | grep <domain>` first.
- **macOS TCC permissions are folder-scoped and per-process.** Folder A working in this session does not mean Folder B will. The reliable fixes are: (a) `/tmp/` (always exempt), (b) Full Disk Access in System Settings (one-time toggle, solves permanently).
- **Starter Render plan is always-on.** Cold-start-from-sleep is NOT a LocoSnap concern. Sentry connect/timeout errors are deploy-churn artifacts during the 30-60s container swap windows, not cold start.
- **Anthropic prompt caching needs the static portion in `system: [{ ..., cache_control: ephemeral }]`.** Sending the entire prompt as `messages[user].content` does NOT trigger caching even if the prompt is mostly static. Each Haiku service was hitting this — fix in `ecc1142`.
- **Polish characters require Write tool, not bash heredoc.** Heredocs do not process Unicode escape sequences correctly. ASS subtitle file with `Ł` written via heredoc renders as escape-sequence text on screen.
- **The brainstorming skill's HARD-GATE applies to every project** including ones that feel small. The 6-question Q&A flow on the leaderboard work surfaced the Pro-tier-gating decision that wasn't on my radar at the start. Process discipline > shortcuts.
