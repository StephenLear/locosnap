## What Worked
- Stripping audio with `-an` on all concat outputs — fixed 12s → 24s padding bug caused by mismatched audio timestamps from mixed screen recording + camera clips
- ASS subtitle `\fscx` scaling to force individual lines to fill screen width
- Moving sprite to `y=0` (top of canvas) to give captions room at bottom — clean separation of content and text
- Blur + snap cut reveal concept (`gblur=sigma=25` → hard cut to clear) — effective tension builder

## What Didn't Work (and Fixes)
- **Emoji in ASS captions** (`🤔`, `⚡`) → showed as yellow boxes. libass can't render colour emoji. Removed and replaced with plain text.
- **Caption size too small** — started at 72px, needed 230px for TikTok. Always start at 150px minimum for 1080×1920.
- **"Move the pixel art up" misread** → moved the caption to top instead of the sprite. Rebuilt with sprite at `y=0`.
- **Mixed ICE types in one video** — ice_hannover.mp4 is ICE 2, long recording scan is ICE 3neo (Class 407). User caught it. Resolved by using 23s recording (ICE 1, Class 401) consistently with ice_hannover.mp4.
- **Wrong station in reveal** — used t=7s from ice_hannover.mp4 for clear snap, which was a different station. Fixed by using t=5s which is the same covered platform as the blurred section.
- **Audio padding in concat** — mixed sources caused 23.88s output from 11.9s of clips. Fixed with `-an`.
- **`eas secret:create` deprecated** → use `eas env:create --scope project --environment production --type string --visibility sensitive/plaintext`

## Lessons Learned & Gotchas
- **libass cannot render colour emoji** — never use emoji in ASS/SRT files. Use text alternatives.
- **TikTok captions: minimum 150px** on 1080×1920. Start big, go bigger.
- **Always `-an` when concatenating mixed sources** — screen recordings + camera footage have mismatched audio timestamps that bloat output duration.
- **Identify clip subjects before building** — state train type and location for each clip, confirm consistency before encoding.
- **"Move X" is ambiguous** — always confirm whether moving content element or caption overlay.
- **ARCHITECTURE.md must be updated** with any new API keys, tokens, tester contacts, or env vars created in a session.
- **`eas env:list`** requires `--environment production` flag or it prompts interactively (fails in non-TTY).
