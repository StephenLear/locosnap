## What Worked
- Tom's response confirmed Android 16 is not the root cause — narrowed to device locale
- Removing expo-localization entirely (not just the plugin) is the correct approach — native module crashes cannot be caught by JS try/catch
- vidIQ video analysis: 1.5-second cliff rule validated the "no text until 2s" decision for the CRH ad
- Two-line stacked caption (big speed + smaller question) over hook footage is the right structure

## What Didn't Work (and Fixes)
- v1.0.9 (remove key prop from Tabs) → wrong root cause entirely; Finnish tester still crashed
- v1.0.10 (remove plugin from app.json) → insufficient; package still installed, native APIs still called at startup
- Audio dropped from crh_c1_hook.mp4 on original cut → fixed by re-cutting with -c:a aac; added null audio to screen recording clips
- EAS build triggered with run_in_background → shell exited before build completed; no APK retrievable. Fixed: re-triggered foreground with timeout:600000
- Caption timestamps not recalculated after scan clip shortened from 5s to 1s → "Even in China" caption would have appeared after video ended; caught and fixed manually

---

## Key Decisions
- **Remove expo-localization entirely, not just plugin**: Removing only the plugin left the JS code still calling native APIs. Full removal is the only safe fix.
- **App defaults to EN on first launch**: Device locale detection removed. User picks language via in-app picker. Simpler and crash-proof.
- **Hold all 18 other testers on v1.0.9**: Correct — any tester with non-EN/DE locale would have crashed. Wait for Finnish tester confirmation of v1.0.11 before wider send.
- **No text until 2s on CRH ad**: Validated by vidIQ "1.5-second cliff" research — visual pattern interrupt first, then text.
- **Music added in TikTok, not baked in**: Algorithm favours native sounds; easier timing control.

---

## Lessons Learned & Gotchas
- A native module crash before Sentry initialises leaves no trace in monitoring. If the app shows a brief background colour then crashes with nothing in Sentry, look at native module initialisation order.
- expo-localization crashes silently on devices with unsupported locales — it is not safe to use in managed Expo workflow if you only support a subset of locales.
- EAS builds must ALWAYS use foreground Bash with timeout:600000. run_in_background exits the shell after upload, before EAS finishes.
- When iterating on crash fixes, state the hypothesis and ask for confirmation before triggering a build. Three builds were needed here; earlier hypothesis verification would have saved time.
- When re-cutting video clips, always run ffprobe on all clips to verify audio streams before final concat.
- When clip lengths change, recalculate ALL caption timestamps — captions at the tail of the video will fall outside the new duration.
