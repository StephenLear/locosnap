## Lessons Learned & Gotchas

- **Class-number verification before recommending a class for an ad.** When proposing a new class subject, Wikipedia-fetch the article BEFORE writing the storyboard or footage research brief. The Taigatrommel/BR 130 + Holzroller/BR 254 confusion both came from training-data conflation that a single Wikipedia fetch would have caught immediately. **NEW HABIT:** for any class-subject pitch, first action is Wikipedia fetch on `de.wikipedia.org/wiki/DR-Baureihe_X` (or similar) and quote the build number / years / nickname back from the article before any further work.
- **DE rail-fan audiences will pick apart any imprecise hook framing.** "Sieht aus wie 1962" worked algorithmically but invited bodyshell-vs-livery pedantry. Future heritage-livery ads should be per-dimension precise: "Lackierung: 1962. Loko: 1996." Avoid hook claims that conflate body design with paint scheme.
- **Worktree node_modules drift is now a 3-session-pattern.** 2026-05-14 handover flagged it. 2026-05-15 handover repeated. Today again — couldn't run tsc from worktree. For any TS work over ~50 LOC, either install node_modules in the worktree once OR work in main checkout and copy to worktree before commit. Below 50 LOC, manual review is acceptable per the prior lesson.
- **`feedback_push_docs_at_session_end.md` rule** added today. Verifying with `git log origin/<branch>..<branch> --oneline` before declaring session close is the operationalisation. Memory claims about git state decay fastest of all memory types.
- **Single-data-point-discipline applies inside the session too.** The 926-view BR 101 v2 early signal is genuinely positive but doesn't equal "the ad worked." 24h read tomorrow is the verdict.
- **Saturday afternoon may be a viable alternative slot** if today's results hold. Add data point — don't conclude.
