- **Scope: vision layer + specs layer + corrections map for all three fixes.** Even when only one layer appeared wrong, all three were reinforced — downstream text generation can re-introduce errors that vision fixed alone won't catch. Same pattern as BR 140 / Sr1-Sr2-Sr3 sessions.
- **No broad `stadler flirt` correction.** Keyed on `sm5` / `vr sm5` only. FLIRT is a worldwide Stadler platform and a global correction would cause collateral damage on Norwegian / German / Swiss / Italian FLIRT scans.
- **Don't reply to "Ehrlich?" on BR 143.** It was between-commenter chatter directed at "iwi finde ich es nd 🥺", not at Stephen. Jumping in would have looked like over-monitoring.
- **Do reply to "iwi finde ich es nd 🥺" with an open question, not a defence.** 🥺 signals soft disappointment, not hostility — the right response is to ask what was missing, not to argue.
- **Honour the public fix-commitment deadline.** User's BR 111 reply said "Fix ist heute Abend live" before the Desiro HC rule existed in code. Shipping it the same evening protects public credibility with undecided German viewers reading the thread.
- **Reply to Oula with Junakalusto name-check.** The Sm5 operator nuance (Junakalusto owns, HSL is customer-facing, VR operates under contract) is exactly the kind of depth that reinforces her valued-bug-reporter position — she'll read it, it signals the fix was done with rigour.

## Lessons Learned & Gotchas
- **BR running numbers masquerade as class names in the prompt output.** "Class 30506", "Class 30777", etc. are AI hallucinations — the real class is whatever design the running number belongs to (Urie S15, King Arthur N15, etc.). The fix is a number-block guard in the prompt, not a per-number override. Same pattern may apply to other railways (e.g. DB 3-digit BRs vs running numbers in the 90xxx range).
- **When a vision rule exists for one sibling but not the other, the model will over-return the known sibling.** The Mireo PRE-FLIGHT CHECK was doing its job — the model correctly identified Mireo-like features — but without a Desiro HC sibling rule to route TO, the model had no way to escape "Mireo" when the train was obviously bilevel. Always pair sibling rules; never fix one in isolation.
- **German TikTok commenters frame paywall irritation as "nutzlos" ("useless").** Different purchase-intent signal than UK/Finnish testers. The 3-lifetime-scan limit is harsh enough that this comment pattern will recur on every DE video until the limit relaxes or in-app messaging makes the cap clearer upfront.
- **Timeline mapping before replying to short German comments.** "Ehrlich?" with no quoted context is ambiguous — map the timestamps of sibling comments in the same thread before assuming direction. The adjacency heuristic (which comment posted right before it) is usually correct.
- **File-based commits (`git commit -F /tmp/commit_msg.txt`) remain the right default.** Today's two commits both used it. No apostrophe/umlaut breakage. The CLAUDE.md rule is holding up — keep defaulting to it.
