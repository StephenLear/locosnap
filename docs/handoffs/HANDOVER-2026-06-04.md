## Lessons Learned & Gotchas
- **Upstash `locosnap` is a GLOBAL (multi-region, eventually-consistent) DB** ‚Äî primary eu-west-1 (Ireland), backend in Frankfurt. Any read-immediately-after-write pattern can hit an unsynced replica ‚Üí null. Blueprint tasks were the victim; anything else with a tight write‚Üíread loop is at risk. The train cache is safe (reads lag writes by minutes+).
- **A backend 404 surfaces in the app as "Couldn't reach LocoSnap servers"** ‚Äî `pollBlueprintStatus` counted any non-2xx as a network error. C now distinguishes 404 (terminal). Keep this in mind when reading future blueprint Sentry events.
- **The "98% cache read" figure is the ANTHROPIC prompt cache, NOT the Redis train cache** ‚Äî don't conflate them (I briefly did). The Redis train cache hit rate is separate (and works ‚Äî confirmed via `[CACHE] HIT` logs).
- **Supabase SQL editor auto-brackets** ‚Äî typing parens/quotes risks mangling. For migration 019 I ran only the bare `ALTER TABLE ‚Ä¶ ADD COLUMN` (no parens/quotes); the COMMENT-ON-COLUMN was skipped (retained in the git file). For complex SQL, paste isn't available via the computer tool ‚Äî keep statements bracket-free or split them.
- **Migration number collision** ‚Äî card-edit took 019; the Social Phase 1 plan originally also said 019 (now corrected to "next available, 020+").
