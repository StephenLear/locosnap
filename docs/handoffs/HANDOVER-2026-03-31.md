## Lessons Learned & Gotchas

- **Tester email on Play beta != in-app signup email** — never assume these match. Always query Supabase auth.users against the tester list to find who has actually signed up and with what email.
- **New profiles don't inherit manual Pro grants** — `UPDATE profiles SET is_pro = true` only affects rows that already exist. A tester who signs up after the grant runs starts on free tier. The monitor handles this automatically now, but it's a gotcha worth remembering.
- **Service role key is not stored locally** — backend `.env` does not exist on this machine. Key must be retrieved from Render → locosnap-backend → Environment Variables → `SUPABASE_SERVICE_KEY`. Keep this in mind for any future Supabase admin operations.
- **RemoteTrigger requires active claude.ai session auth** — if it returns 401, fall back to `mcp__scheduled-tasks` for local scheduling. Both tools are available but serve different use cases (remote = runs in cloud without local machine; local = requires machine to be on).
