## Key Decisions
- **Use Wikidata + AI hybrid, not AI-only**: AI memory unreliable for precise specs (voltage, length, formation). Wikidata wins for factual fields, AI fills narrative/context gaps.
- **Don't switch AI providers (OpenAI vs Claude)**: Hallucination is an architecture problem, not a model problem. Both models unreliable for precise engineering specs from memory.
- **Name-first Wikidata disambiguation**: Simpler than year-based filtering, works for most cases where the vision model returns a meaningful train name.
- **New ICE 3 TikTok video**: Existing German clips (Taurus/ES 64 U2) already used — intentionally different train for variety.

## Lessons Learned & Gotchas
- **Android `rotateY` without `perspective` crashes** — always add `{ perspective: 1000 }` before any rotateY in React Native transform arrays
- **Wikidata class names are ambiguous** — "Class 403" can mean completely different trains in different eras. Always search by name first when available.
- **Finnish/Nordic Wikidata coverage is sparse** — Sr1, Dv12 entries may be incomplete or missing key fields. AI fallback for these classes is weaker than for German/UK stock.
- **TikTok hook must work in first 0.5 seconds** — analytics showed 25% retention, most dropped at 0:01. First frame needs text overlay.
- **Pixabay "ice train" results include iStock sponsored items at top** — scroll past to free royalty-free section
- **Wikimedia Commons has good free ICE footage** — `Einfahrender_ICE_in_Hannover.ogv` (CC BY-SA 4.0, 34s, 480x640)
