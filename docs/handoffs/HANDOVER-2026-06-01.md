## What Didn't Work (and Fixes)
- **Two-stage split regressed accuracy.** Lean core = 27/36 vs 28/36; SW1001 (Mendip) went correct→"Class 08" (Steph's fixed bug). → **Reverted.** Why it can't work: (a) a focused-prompt PoC returned the IDENTICAL misIDs as the 85K prompt, so prompt SIZE is not the cause; (b) stage-2 can't recover a removed load-bearing rule — stage-1 returns the wrong class, which never triggers the right family lookup. Documented in CHANGELOG + `project_ai_cost_baseline` memory: **do not re-attempt.**
- **In-place misID fixes don't take.** Adding guards to the 85K prompt for BR159/Mireo/SW1001 changed nothing (temp 0, identical wrong output). These are model/image limits.
- **Anthropic key wouldn't load** for the eval: dotenv won't override an env var already set empty in the shell. → Run with `ANTHROPIC_API_KEY=$(grep '^ANTHROPIC_API_KEY=' .env | cut -d= -f2-) npm run vision:eval`.
- **Bash sandbox blocked the `.env` write + external-dir reads + API calls.** → Used `dangerouslyDisableSandbox: true` for the key write and eval runs.
- **Render dashboard needed login** (Anthropic/Supabase were already authed in the same browser; Render wasn't). User logged in, then it worked.

## Lessons Learned & Gotchas
- **Don't deploy while testers are actively scanning** — the `517eb26` restart knocked out Leon's blueprint mid-use. Render restart = brief unreachability + kills in-flight (in-memory) blueprint jobs.
- **A focused prompt is accuracy-neutral vs the bloated one** (PoC) — so prompt trims won't fix misIDs, and the only safe cost lever there is caching, not restructuring.
- **The 8 known misIDs are model/image limits** (ICE4→408, BR159→193, Mireo→563, Thumper→411, SW1001→SW1500/Class08, Newag Dragon→Griffin, ICE3neo→403, ÖBB5022→4024) — need better/closer images or fleet-number reliance, NOT prompt edits.
