## Lessons Learned & Gotchas

- **Stale backlog entries can hide already-shipped features.** #12 was tagged Medium priority "user agreed 2026-04-23" but the flip animation + compare mode were already shipped by then. The 3-day-old backlog memory wasn't accurate. Pattern for future: audit current code state BEFORE scoping a backlog item, not after.
- **`MAX_HISTORY = 200` was a silent ceiling for active testers.** Steph has 210 spots / 85 unique — fell off the local view at the boundary. Other heavy users will hit this too. v1.0.31 bumps to 1000; v1.0.32+ should ship true pagination.
- **Render auto-deploys fired on every push to main regardless of path** until today's buildFilter. Pattern was 2× confirmed (yesterday's backend ships + today's doc commits). Now structurally fixed.
- **Sentry source-map upload was already running for Android builds via `@sentry/react-native`'s Gradle plugin** — `:app:createBundleReleaseJsAndAssets_SentryUpload_*` task fires when `SENTRY_AUTH_TOKEN` + `SENTRY_ORG` + `SENTRY_PROJECT` are present at build time. EAS Secrets pane is the place to verify; `eas env:list production --format=long` shows them. No code change was needed to make symbolication work — it was already working. The "no deobfuscation file" Play Console warning was independent.
- **R8 minification is OFF by default for Expo SDK 54 managed-workflow Android production builds.** APK is larger than necessary; no `mapping.txt` is generated. Enabling via `expo-build-properties` is the supported path — no need to eject or prebuild.
- **Google Play promo codes are free-only.** Cannot do paid intro offer via promo code. Use a **subscription offer with Single payment phase** on the base plan instead (eligibility: "New customer acquisition"). The "Discounted recurring payment" type requires ≥ 2 billing periods so it's not usable for "1 month at £1".
- **`expo install <package>` adds the plugin as a bare string** at the end of the `plugins` array. To add config, manually convert to array form `[ "name", { ...config } ]`.
- **Cold creator DM with specific, narrow ask converts fast.** Samuel: "2-3 second clips, full credit, link in description" — said yes in writing within hours. Compare to vague partnership inbound DMs that usually go nowhere.
- **TikTok Studio comment search** finds engagements by username — definitive way to check whether a user has commented on any LocoSnap content. URL: `/tiktokstudio/comment` with search top-right.
- **Apple App Store conversion rate of 12.5%** is genuinely top-decile for the App Store (industry median ~2-5%). The bottleneck is impressions, not optimization. Frequent posting compounds at 2-4× normal rates.
- **The `__mocks__/analytics.ts` stub + `jest.config.js moduleNameMapper`** is the standard pattern for unblocking tests that transitively import a Sentry-using service. Don't add `transformIgnorePatterns` for `@sentry/react-native` — too much surface area; stub the consumer module instead.
- **Heredoc commit messages still risky** for messages with apostrophes / em-dashes / special chars. File-based via `/tmp/commit_msg.txt` + `git commit -F` worked cleanly all session.
- **App Store Connect & Play Console release notes can't be set via `eas submit`** — must be pasted manually per locale (EN + DE) after the submission goes through. Bug-fix-first, then non-financial features, then financial offers (per `feedback_release_notes_order.md`).
- **iOS release notes must NOT include the £1 first-month offer** — Android-only, no Apple equivalent set up. Per skill `app-store-release-notes` Apple-rejects platform-mismatched copy.
