- **Welcome email order: DE → EN → PL.** DE is #1 market per spec doc and per `project_market_focus.md`.
- **No founder CC on automated sends.** Per-signup CC would flood inbox at scale. Reply-To `hello@locosnap.app` still routes responses to Proton via ImprovMX.
- **ICE fix: hardcode specs, not tune vision.** Vision rules in `vision.ts` already comprehensive (60+ lines covering car-counts, nose shapes, Schaku coupler flap discriminator, BR 412 default with positive-evidence requirement). Specs-layer override locks correct values even when vision drifts.

- **Vision-side 401/402 disambiguation deferred.** Existing pre-flight check is thorough; further iteration is low-yield without ≥3 fresh misID examples to drive specific rule changes.
- **Weight threshold of >= 5 tonnes.** Conservative floor — Schienenbus single railcar is ~14 t, smallest draisines ~6 t. Anything under 5 t is hallucination.
- **Backfill cohort split by week, not in one blast.** Smaller batches let us validate template / change copy / catch typo patterns before committing to the next slice. Final cumulative: 67 → 79 → 81 → 75 = 302 across 4 batches.

## Lessons Learned & Gotchas

- **Read directory contents before relying on memory** — `docs/email-welcome-spec.md` existed and had approved copy from a parallel session. If I'd just barreled into building, would have shipped a copy mismatch with the spec doc.
- **Chrome MCP domain blocks** — TikTok, Supabase blocked. WebFetch also fails on JS-rendered SPAs like TikTok. For those: ask user for screenshot paste or walk them through manually with copy-paste-ready instructions (SQL, click sequences, etc).
- **Supabase Database Webhooks hide `auth` schema** — can't trigger on `auth.users` INSERT from the Dashboard UI. Use `net.http_post` in a Postgres trigger via SQL editor instead. Cleaner anyway (no Dashboard dependency, version-controlled).
- **"1 tonnes" / `parseFloat("1") > 0` was the old weight validation** — only caught null/missing. New `>= 5` threshold catches AI hallucinations on weight.
- **npm node_modules don't sync between worktrees** — main worktree needed `npm install` after worktree branch installed resend
- **Cache version bumps need ARCHITECTURE.md log entries** — established convention (v7→v8, v8→v9 both recorded in header). Easy to miss; Review 2 caught it.
- **AI weight hallucinations are systemic, not class-specific** — the `1 tonnes` bug could hit any class. Threshold validation is belt-and-braces for any future class we forget to hardcode.
- **Class string format variants** — vision returns class strings in 5-8 different formats per class ("BR 401", "br401", "401", "DB 401", "DB BR 401", "Baureihe 401", "ICE 1", "ice1"). Every hardcoded spec needs all these alias keys.
- **Two-review rule actually catches things** — wasn't ceremonial. ICE L stub miss was a real bug that would have hit a tester eventually.
- **Resend's API success ≠ delivery success.** API returns 200 + message_id on suppressed addresses but never delivers. Dashboard export's `last_event` column is the truth. Cross-reference for any non-trivial batch send.
- **Supabase Auth OTP shares Resend quota.** Sign-in emails ("Your LocoSnap Sign-In Code") flow through the same Resend account with a Supabase-owned API key. When sizing future batches, account for background OTP traffic on top of intentional sends.
- **Typo-domain emails are an actual systemic issue.** Across 4 batches and 380 total addresses surveyed: 8 had bad domains (~2%). Pattern detection (gnail, gmaio, outloo, oulook, hitmail, trailing-s, missing-TLD) catches most. Same-person duplicates with corrected variants are common — saves us from missing anyone.
- **Resend suppression list is per-account, persistent.** Both addresses suppressed in batch 2 had historical bounce/complaint history that we couldn't see ahead of time. Can't manually un-suppress without going into the dashboard one-by-one. Better to assume any modern Resend account has some suppression baggage and verify post-send.
- **Permission-check footage BEFORE building.** ARI watermark on the original Munich clip would have triggered a takedown like BR 140 v5 / Bahnen in Ostbayern (2026-05-10). Frame-sampling caught it on the first read — never trust a filename or thumbnail; always look at actual frames before assuming clearance.
- **Year anchor on heritage-livery ads ties to origin, not arbitrary mid-life dates.** "1972" for BR 420 maps to the iconic Olympic year and the kieselgrau/orange livery's purpose. "1993" was an arbitrary mid-life year only relevant when the Schmierfett-Affäre clip was the source. When sourcing changes, the anchor changes too — don't carry over a year from a discarded narrative.
- **Sentinel + Thomas Hill + Hunslet weren't the answer.** First-pass instinct on a yellow industrial quarry shunter pointed at the UK industrial-builder usual suspects. Actual answer was an American GM EMD SW1001 — only one operational example in the UK, the unsung loco that made the famous Class 59s happen at Foster Yeoman. Two web searches resolved it.
- **Steph's misID-to-fix loop is now 6-deep.** Class 45 (May 9) / Class 57 / Class 59 / Class 70 / Class 11 / Class 52 / Class 14 / Class 14 vs 31 / now SW1001 (May 18). She's the deepest tester evangelist in the project and routinely surfaces things vision can't catch from prior training. Every reply that names the actual class with story-attached context (e.g. SW1001 as Class 59 ancestor) reinforces her engagement.
