Critically, this SW1001's reliability hauling 2,500-ton stone trains is what convinced Foster Yeoman to go back to EMD a few years later and order the Class 59s — making it the *direct ancestor* of the famous Class 59 fleet at Mendip Rail. The unsung loco that made the Class 59s happen.

## Key Decisions

- **Welcome email format: hybrid** (HTML wrapper with logo + plain-text fallback). Spec doc said plain text; user chose HTML. Both shipped.
- **Welcome email order: DE → EN → PL.** DE is #1 market per spec doc and per `project_market_focus.md`.

- **ICE fix: hardcode specs, not tune vision.** Vision rules in `vision.ts` already comprehensive (60+ lines covering car-counts, nose shapes, Schaku coupler flap discriminator, BR 412 default with positive-evidence requirement). Specs-layer override locks correct values even when vision drifts.
- **Direct commits to main for tester batches.** Matches `cec1f13`, `14a1b37`, `ee21ed6` precedent. Render auto-deploys.
- **Backfill to ~470 non-Pro users deferred.** Ship + observe per-signup pipeline for 1-2 days, then blast. Two reasons: (a) verify real-inbox delivery across email clients first, (b) don't blast hundreds with a template that turns out to render badly.
- **Vision-side 401/402 disambiguation deferred.** Existing pre-flight check is thorough; further iteration is low-yield without ≥3 fresh misID examples to drive specific rule changes.
- **Weight threshold of >= 5 tonnes.** Conservative floor — Schienenbus single railcar is ~14 t, smallest draisines ~6 t. Anything under 5 t is hallucination.

## Lessons Learned & Gotchas

- **Read directory contents before relying on memory** — `docs/email-welcome-spec.md` existed and had approved copy from a parallel session. If I'd just barreled into building, would have shipped a copy mismatch with the spec doc.
- **Chrome MCP domain blocks** — TikTok, Supabase blocked. WebFetch also fails on JS-rendered SPAs like TikTok. For those: ask user for screenshot paste or walk them through manually with copy-paste-ready instructions (SQL, click sequences, etc).
- **Supabase Database Webhooks hide `auth` schema** — can't trigger on `auth.users` INSERT from the Dashboard UI. Use `net.http_post` in a Postgres trigger via SQL editor instead. Cleaner anyway (no Dashboard dependency, version-controlled).
- **"1 tonnes" / `parseFloat("1") > 0` was the old weight validation** — only caught null/missing. New `>= 5` threshold catches AI hallucinations on weight.
- **npm node_modules don't sync between worktrees** — main worktree needed `npm install` after worktree branch installed resend
- **Cache version bumps need ARCHITECTURE.md log entries** — established convention (v7→v8, v8→v9 both recorded in header). Easy to miss; Review 2 caught it.
- **AI weight hallucinations are systemic, not class-specific** — the `1 tonnes` bug could hit any class. Threshold validation is belt-and-braces for any future class we forget to hardcode.
- **Class string format variants** — vision returns class strings in 5-8 different formats per class ("BR 401", "br401", "401", "DB 401", "DB BR 401", "Baureihe 401", "ICE 1", "ice1"). Every hardcoded spec needs all these alias keys.
- **Two-review rule actually catches things** — wasn't ceremonial. ICE L stub miss was a real bug that would have hit a tester eventually.
