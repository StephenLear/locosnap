## Lessons Learned & Gotchas

- **Before attributing a reported bug to a recent code change, confirm the user's app version.** Particularly for Android where Play rollout state and rejection state can mean the user is one or even two versions behind main. The "is this from yesterday's commit" instinct is fast but unreliable across the iOS-vs-Android-vs-rejected-build matrix.
- **Auto-charge state is ambient, not a fire.** Document infrastructure billing state explicitly (with the date) so it doesn't get re-raised as urgent every time the dashboard shows a low balance.
- **Profile screen stats in v1.0.34 are 100% local Zustand-derived.** No server query for Total spots / Unique Classes / Rarest Find / Favourite Operator. If `loadHistory()` skips the authenticated branch (session expired, transient network, etc.), the screen renders from AsyncStorage with whatever local cache exists. This is an architectural choice not a bug, but it interacts badly with silent Supabase error swallowing.
- **DeepSeek has no vision model.** Cheap text pricing is structurally irrelevant for vision-pipeline apps. Worth a quarterly recheck — if they ship vision, the cost equation changes by an order of magnitude.
- **Gemini AI Overview / Lens is what users compare us to in price-friction comments,** not Gemini API. Consumer pricing comparison (LocoSnap €2.99 vs Gemini Advanced ~$20) is the relevant frame, not raw token economics.
- **Play Console Data safety form expects you to declare per-SDK data collection.** PostHog distinct_id, Sentry device info, RC advertising ID — all count as "Device or other IDs" + "Diagnostics" + "App activity" + "Crash logs" depending on the SDK. Future builds with new SDKs (e.g. an Adjust / AppsFlyer integration if ever) need a parallel form audit.
- **Plain-English / physical-world analogy beats jargon when comprehension stalls.** "Notepad on phone vs filing cabinet on server" carried more weight than five paragraphs of Zustand/JWT/loadHistory detail. Default to the analogy first when user signals confusion.
- **`feedback_handover_single_file_per_day.md` rule applied** — today's handover written as `HANDOVER-2026-05-27.md`, not incremented. Single file per work day.
