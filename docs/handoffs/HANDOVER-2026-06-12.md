## Lessons Learned & Gotchas
- **DATE/CLOCK DISCREPANCY (important):** this session operated throughout as if it were 06-12 (Friday), and the commits are all 06-12 morning — but the real clock by session end reads **2026-06-13 (Saturday) ~18:46 local**. This means the ad schedule timing needs a sanity check (see Next Steps #1): EN57 was slotted for "Saturday" and it is now Saturday evening.
- **Blocking on TikTok removes the blocked user's comments AND every reply nested under them.** Weigh this before blocking anyone whose thread contains an exchange worth keeping (here the Mathis defence + our reply vanished with Lukas). If the buried sub-thread matters, answer-and-leave rather than block.
- **Class-anchored rarity, reinforced**: when an ad drives scan traffic to a class, lock the tier AND invalidate the cache together (the EN57 lesson, now applied to ICE 1). An unlocked tier is operator-swayable and viewer scans won't match the ad's card.
- **Cache-invalidation timestamps on ad days must be just-past-deploy, not end-of-day** — an end-of-day timestamp makes every ad-driven scan a cache miss until midnight.
- **Stale "awaiting push" memory claims decay fastest of all** — always `git log origin/main..HEAD` and/or `git branch -r --contains <sha>` before trusting any "not yet deployed / commit pending" note. Caught the BR 140 claim this way.
