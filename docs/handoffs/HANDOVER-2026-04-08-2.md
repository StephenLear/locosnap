## What Worked

- Parallel clip cutting — all 4 steam videos cut simultaneously, saving time
- Frame extraction workflow for identifying screen recording card reveal timestamps
- drawtext filter with explicit fontfile path — reliable on macOS, unlike ASS/libass
- EAS autoIncrement for iOS buildNumber — no manual management needed

## What Didn't Work (and Fixes)

- **Read tool on .MOV file** -> binary error, then said "can't preview" -> user corrected -> extracted frames with ffmpeg. Skill rule added: never use Read on video files.
- **ASS subtitle rendering** -> 3 failed iterations (font not found by libass, PTS timing conflict with `-ss` seek) -> fell back to drawtext which worked first time. Skill rule added: prefer drawtext on macOS.
- **Text clipping at 165px and 110px** -> "Destroyed in 2021" and German "Wiederaufgebaut 2025" clipped at edges -> reduced font size and shortened text. Skill rule added: character count check before choosing fontsize.
- **Used "AI" instead of "LocoSnap" in draft captions** -> user corrected: always say "LocoSnap" not "AI" for branding. Skill rule added.

## Key Decisions

- **Bilingual as separate videos, not mixed**: Research showed mixing languages in one video dilutes algorithmic reach. Post English first, German ~1 hour later on same or different platform.
- **iOS version stays at 1.0.17**: Version in app.json is shared across platforms. Apple accepts non-sequential version jumps. Can't set iOS to 1.0.8 without breaking Android at 1.0.17.
- **Polish content deprioritised**: Zero downloads on 2026-04-07/08 despite follower growth from Polish TikToks. UK and Germany are the conversion markets (iOS-heavy, App Store accessible).
- **Steam footage from YouTube accepted**: User confirmed footage is not copyrighted. Built with LocoSnap card reveals and end screens.
- **"Not 100% Sure" (65% confidence) used as feature**: The Earl's low confidence result is framed positively in the caption — "LocoSnap wasn't sure" creates curiosity and drives comments.

## Lessons Learned & Gotchas

- On macOS, always use `drawtext` with `fontfile=/System/Library/Fonts/Supplemental/Arial\ Black.ttf` — ASS/libass font discovery is unreliable
- At 1080px width with Arial Black: 110px fits ~14 chars, 90px fits ~17 chars, 70px fits ~22 chars — check before rendering
- Screen recording from iPhone ProMax (1206x2622) needs `scale=1080:2348,crop=1080:1920:0:214` to fit TikTok portrait
- EAS `appVersionSource: "remote"` + `autoIncrement: true` handles iOS buildNumber automatically — no manual config needed
- The ET22 max speed in the handover (HANDOVER-2026-04-08.md) said "140 km/h" but the actual prompt had "125 km/h" — always verify by reading the file, not trusting handover values
