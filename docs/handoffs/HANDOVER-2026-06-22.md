## What Didn't Work (and Fixes)
- **Transparent Marker tap targets → Android default red pins.** A transparent custom marker view doesn't suppress Android's default pin → replaced with a visible rarity-coloured dot + `tracksViewChanges` toggle.
- **Place name always "This area".** First broadened `reverseGeocodeAsync` fields (insufficient — device geocoder returns nothing on Android) → switched to BigDataCloud.
- **New APK "updated" but kept old code.** EAS preview builds without `autoIncrement` reuse the version code, so Android silently kept the old build. Fix: uninstall-then-install (one-time), then added `autoIncrement` so future installs go in place. Diagnostic tell: the Atlas default grid (10 km = old build, 25 km = new build).
- **Maps key duplicate-key tsc error.** `"stadler flirt 3"` already existed in `WIKIDATA_CORRECTIONS` (BR 428 full specs) → dropped the duplicate, kept the existing one.
- **GCP/Play Console SPA deep-links bounce to home** — navigate via UI clicks or retry the deep link (it lands on the second try).

## Lessons Learned & Gotchas
- **Atlas place names depend on BigDataCloud** (external, keyless) — `expo-location` reverse-geocode is dead on Android. Don't revert.
- **The Maps key needs BOTH SHA-1s** — EAS keystore (test builds) AND Play App Signing (production). Both now registered. If the live Android map ever goes blank, check the key's Android restrictions first.
- **EAS preview needs `autoIncrement`** (now set) or Android won't update in place.
- **read IG and TikTok separately for the same ad** — the BR 101 gacha "died at 0:01" on TikTok but held a healthy retention curve on IG. (Also: the gacha format is retired in favour of tier-debate.)
- **Maps key is unrestricted-dangerous in a PUBLIC repo until locked** — it's now signature-restricted, so safe.
- **Every ad caption MUST lead its hashtags with `#LocoSnap`** — Stephen flagged it kept being dropped from the recent gacha/tier-debate captions. Brand search was ~50% of launch inbound, so omitting it costs reach. New memory `feedback_always_include_locosnap_hashtag`; self-check before showing any caption.
- **Don't scan an OLD AD to generate a card** — Stephen's BR 111 re-scan was run on the old BR 111 ad video, so its burned-in "Seit 1974" hook text bled into the card image (had to mask it). Scan a plain photo for a clean card.
