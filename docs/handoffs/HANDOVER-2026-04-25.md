## Lessons Learned & Gotchas

- **Forbid lists fail without a positive anchor.** When the model has zero training data on a class (e.g. Class 69, post-2021), every "do NOT call this Class X" guard just pushes the model to the next-most-plausible familiar class. The fix is to give the model something to lock onto (livery, fleet number range, distinctive visual cue) — not more things to avoid. Three-strikes case proved this; positive anchor cracked it.
- **Same-day tester loops compound.** Oula gave us the exact diagnostic key for Sm2 v2.1 (the trailer-car numbering scheme) because he saw us deploy v2 within hours of his original report and trusted that v2.1 would land same-day. Tester-evangelist pattern from `memory/project_tester_evangelist_pattern.md` extended again — Oula's case is a second data point.
- **Model migration housekeeping is two separate things.** Sonnet 4 → Sonnet 4.6 = same-price quality upgrade (do this freely). Sonnet → Haiku = cost-saving downgrade (only safe for structured-JSON tasks, not for vision). Easy to conflate when explaining to the user — keep them separate in any cost discussion.
- **Letterbox is wrong for LocoSnap landscape→portrait conversion.** The video-editing skill mentions black-bar letterbox as a pattern but for action footage with subjects spanning the horizontal frame, blur-fill background is the correct choice (TikTok-standard). Use `scale=W:H:force_original_aspect_ratio=increase,crop=W:H,boxblur=30:1` for background, scaled-to-fit foreground overlaid centered.
- **Verify video frame composition before encoding the full 10s.** First BR 430 ad shipped with awful black bars to the user before any frame extraction was done by the user. The skill's "extract verification frame before encoding" rule should fire on the FIRST encode of any new pattern, not just on text overlays.
- **Sentry alerts from emulator builds are Play pre-launch robo-test noise.** Look for `os.build` containing `eng.ubuntu` / `sdk_phone_arm64-eng`, `isSideLoaded: true`, and a US-datacenter geography (Boardman = AWS us-west-2 / Google Cloud). Archive "until escalating" — they're not real users.
- **`docs/handoffs/`, `docs/plans/`, `docs/design/` continue accumulating untracked.** 13+ handover files and 2 plan files are on disk but not in git. Not blocking anything but worth a `git add docs/ && git commit` whenever there's a free moment. Same flag as 2026-04-24 handover.
- **Sm-class fleet numbering is a known German/Finnish-rail feature, not a quirk.** Multiple-unit sets often have separate ranges for powered cars vs trailers (powered + 200 in the Sm2 case). Worth checking before writing future fleet-number rules — same risk could apply to Sm4 (6301–6330 — but only 30 sets, so unclear if there's a trailer offset; verify if Sm4 ever misclassifies).
- **The CLAUDE.md rule "After any backend commit — push-or-hold decision required" worked correctly all five times today.** Each commit explicitly stated "Not yet deployed — needs a push to go live on Render" before pushing. Rule preserved.
