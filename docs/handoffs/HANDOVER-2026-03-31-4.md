## What Worked
- The two-step ICE generation pre-flight (generation first, then sub-class) is the correct structure — previous flat list caused ICE 1 to fall through.
- Cab nose shape as primary NS discriminator: front-on photos can't show double vs single deck, but the nose profile (pointed V vs flat rectangular) IS visible from the front.
- The b-roll clip (NSI ICNG-B) showing ICNG arriving alongside a VIRM at Rotterdam CS is the strongest piece of footage — old vs new contrast in one shot.
- FFmpeg `-an` flag on all clips to avoid audio timestamp mismatch on concat.
- Checking frames with Read tool before finalising to catch text overflow and positioning issues early.

## What Didn't Work (and Fixes)
- **VT 650 misidentification** — model pattern-matched "modern rounded cab + yellow" to VT 650. Fixed by adding CRITICAL EXCLUSION to the VT 650 rule and the NS pre-flight check.
- **"Could not identify" (not_a_train) on second scan** — model had no confident NS match and no strong prior for ICNG. Fixed by the NS pre-flight check firing early before the model reaches the long German/UK disambiguation list.
- **VIRM still returned after first fix** — first NS rule relied on side-window count to distinguish VIRM from ICNG, which doesn't work on front-on shots. Fixed by switching primary discriminator to cab nose shape.
- **Text clipping (v2)** — "Newest intercity in the Netherlands." was 95px Impact and exceeded 1080px frame width. Fixed by splitting onto three lines at 110px.
- **CTA overlapping app buttons (v4/v5)** — "LocoSnap — link in bio" at y=1580 sat on the Save/Share/Details buttons. Fixed by moving to y=1530 which covers "Tap to flip" (user confirmed acceptable) and clears the buttons.
- **Screen recording aspect ratio** — source is 1206x2622 (not 9:16). Required `crop=1080:1920:63:351` to centre-crop to 1080x1920 before processing.

## Key Decisions
- **Three separate prompt commits rather than one** — each scan test revealed a new failure mode; iterative fixes with tests between each one.
- **Cab nose shape as primary ICNG discriminator** — more reliable than side-window count because the vast majority of ICNG photos are front-on station shots.
- **locosnap_icng_v7.mp4 as final** — v6 used 85px CTA (inconsistent with rest of video), v7 restored 105px with same y=1530 position.
- **No audio in exported video** — add music natively in TikTok to avoid re-encoding and to allow different tracks per platform.

---

## Lessons Learned & Gotchas
- Vision prompt rules buried deep in a long list have weak effect — pre-flight checks (prominent, at the top) are far more reliable for known problem classes.
- The ICNG has limited AI training data (entered service April 2023). Prompt text describing visual features can compensate but only if it fires early enough.
- The VT 650 rule had no negative exclusion. Any rule that says "if it looks like X, return X" needs a corresponding "but NOT if Y" for the most common false positives.
- Screen recording crop offset: original is 1206x2622, crop starts at x=63, y=351. Any y-coordinate placed in the cropped frame is 351px less than the equivalent position in the original recording.
- `ffprobe` is at `/opt/homebrew/bin/ffprobe`, not on PATH by default in Claude Code sessions. Always use full path.
