## Lessons Learned & Gotchas
- **A class can have correct KNOWN_SPECS but no vision rule** → it keeps getting misID'd (Class 59 drifted 66→60 over a month). When a class recurs as a misID, check the VISION layer, not just specs.
- **Class-anchored rarity needs a matching CLASS_INVALIDATIONS entry** — locking a tier without invalidating leaves stale cached tiers being served (the 06-05 EN57 gap).
- **The leaderboard counts `distinct train_id`** — so operator fragmentation inflates unique-class counts, and the 021 merge will *reduce* them. Any trains-row merge is leaderboard-visible.
- **Social fetchers/migration ordering** — the frontend ships graceful-degrading (safe), but the feature is dead until migration 020 is applied. Don't claim it "works" before 020.
