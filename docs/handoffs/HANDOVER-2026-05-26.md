### Proper fix — migration 017 + frontend language-sync (commit `117dd6b`)

**Migration 017** (`supabase/migrations/017_profiles_language.sql`):
- ADD COLUMN `language text` nullable so backfill could run first
- Backfill from `country_code` heuristic (idempotent — skip already-set rows): DE/AT/CH → de, PL → pl, else en
- ALTER COLUMN SET DEFAULT 'en' (covers Supabase Auth profile-create trigger + any future INSERT that omits language)
- ALTER COLUMN SET NOT NULL (safe — backfill populated every row)
- ADD CONSTRAINT CHECK `language in ('en', 'de', 'pl')` (DB-layer protection against typo writes)
- Audit-compliant per `feedback_supabase_silent_persistence_failures.md`

## Key Decisions

- **Merge commit (not squash) for the v1.0.35 PR** — preserves per-phase boundary in main's history for blame/bisect later. Trade-off: less tidy single-line main history; acceptable for a release this size.
- **Migration 017 backfilled from country_code instead of leaving NULL.** Existing users get a sensible default (DE/AT/CH→de, PL→pl, else en) without waiting for them to open the v1.0.35 build. Frontend sync overrides on next AsyncStorage divergence.
- **CHECK constraint on profiles.language at the DB layer** — blocks frontend bugs from poisoning the column with typo writes (e.g. `'EN'` capital, `'german'` literal). Belt-and-braces with the TypeScript interface narrowing.
- **NetInfo dependency NOT added for Phase E offline queue** — AppState + lazy in-saveSpot flush cover the common case. Can layer NetInfo on later if Sentry signals indicate the queue accumulating without flushing.
- **3-beat ad structure (no card-reveal beat)** — once the BR 232 ScreenRecording bug was discovered, decided to drop the in-app card-reveal beat entirely rather than re-record. Spotify / Netflix / Disney+ price-led ads don't show app UI; the price is the hook, the trains are the proof of breadth, the value is inferred. Cleaner format.
- **All-yellow endcard text via drawbox + drawtext overlay** — chosen over rebuilding the endcards from scratch with the logo image. Faster (10 min per endcard) and works with all three existing endcard layouts identically.
- **DE today, PL Wednesday, UK Thursday** — 24h gaps between launches so each cut gets a clean signal-vs-noise read in its market. Same pattern as last week's launch ad cycle.
- **Permanently delete `~/Projects/package.json.bak` + `~/Projects/package-lock.json`** rather than leave as `.bak` — they're confirmed cruft from a months-old accidental npm install, no value retained.
- **Submit Apple build with EULA verified + Phase C dismissability screenshot in review notes** — preempts the most likely Apple §7 rejection vector for an auto-open paywall surface.

## Lessons Learned & Gotchas

- **Schema audit before claiming a feature is ready.** Phase G shipped against a column that didn't exist. The audit rule has been in memory since 2026-05-05 (`feedback_migration_column_audit.md`); apply it on every `<table>.<col>` reference, not just on UPDATE/INSERT statements.
- **Stray monorepo-root package.json is a real EAS gotcha.** EAS walks up from the working directory looking for the project root and can be fooled by orphan package.jsons several levels above. Worth a quick `ls ~/Projects/package.json` check before any new EAS build.
- **Apple intro offers are NOT App-Review-gated.** They flip active automatically on the configured start date — "upcoming change 26/5" → active during 26/5 without separate Apple review. My earlier "submit for review" framing was wrong; corrected when user pointed out App Store Connect just showed "upcoming change", not "for review".
- **Polish "ST22" was actually ET22.** The user's existing PL ad uses both `#ET22` and `#ST22` as SEO hashtags. ST22 isn't a real PKP class — confirmed via TikTok oEmbed metadata of the user's own video. Saved confusion: ET22 + Pendolino is the correct PL old/new pair, abundant footage available, Dragon was the alternate "evolution" pair (rejected in favour of the contrast pair).
- **Render Cron Job's default Language is auto-detected Python 3 in some flows.** Must explicitly switch to Node for our backend, otherwise the build fails immediately with `Cannot find module 'typescript'`. Worth documenting in any future cron walkthrough.
- **The TikTok 5-hashtag cap doesn't apply to Instagram.** IG sweet spot is 10-15 hashtags; spamming all 30 is no longer the meta. Documented full IG hashtag sets per market for this campaign.
- **The drawbox + drawtext technique works cleanly for in-place endcard text-color changes** without needing to rebuild the endcard from source assets. Black rectangles over original text + new yellow drawtext = same layout, different palette. Reusable for any future text-color tweak on stale endcards.
