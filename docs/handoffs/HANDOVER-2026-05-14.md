## Lessons Learned & Gotchas

- **READ MEMORY BEFORE ASKING GATING QUESTIONS.** Critical lesson surfaced this session: when Stephen flagged Apple `pro_lifetime` approval, I asked four questions about RC config / Play status / paywall surfacing — every one of which was already answered in `project_revenuecat_topology.md` and `project_lifetime_pro_demand.md`. He had to explicitly say "your memory from a few days ago" before I read. Forward rule (already in memory): **when user mentions any product/SKU/RC/Play/Apple state change, read the relevant memory file FIRST, then ask only the unanswered questions.** Memory hygiene applies to recall as well as writing.
- **`eas submit --non-interactive 2>&1 | tail -30` buffers all output until EOF.** Submit can be running fine but invisible for 20+ min. Use `--verbose` (still buffered through `tail` though), `tee` to a file, or background without `tail`. For future submits: pipe to file with `tee /tmp/eas-submit.log` so progress is visible to `cat` mid-flight.
- **Main repo node_modules drifts from package.json after worktree-to-main pushes.** Worktree commits land via push; main repo working tree never `npm install`s them. Before any EAS build from main, run `npm install` in `frontend/`. Save the 1-2 lost minutes from a failed first attempt.
- **R8 minification on a React Native + Expo app cuts ~6 MB / 22% of install size.** New baseline for LocoSnap Android: 22.2 MB new-install / 7.2 MB update. Worth pasting into the ARCHITECTURE doc's distribution metrics block as the v1.0.31+ baseline. If a future version sees install size creep back up, R8 has regressed (look at `proguard-rules.pro` keep-rules or new dependencies pulling in heavy reflection paths).
- **Worktree absolute paths to project root go to MAIN working tree, not the worktree** (carried lesson from 2026-05-13). All `Edit`/`Write` calls in this session used `/Users/StephenLear/Projects/locosnap/docs/...` directly, which resolved to the main repo — that's actually what I wanted (push needs to come from main), but worth re-stating: the worktree is for command isolation, the file edits hit the main checkout.
- **TikTok video pin state is third-party platform state, not memory state.** Memory tracks performance + audience but not pin/unpin actions. Verify before recommending "pin this video" or "rerun this ad" — ask or check the dashboard.
- **ASC notarisation variance: 5-30 min is normal.** Today's 24 min was at the upper end but not concerning. If it crosses 45 min with no submit-success log line, kill + re-run with `--verbose` to see where it's hung.
- **Day-1 0.8% / Day-7 1.4% / Day-35 2.3% iOS conversion** is the funnel-bleed problem the v1.0.31 soft-prompt ladder targets. Watch these three numbers post-v1.0.31-rollout: any movement on Day-1 specifically validates the scan-2/4/5 soft-prompts; movement on Day-35 without Day-7 lift = soft-prompts aren't converting on the first session but might be priming later sub starts.
