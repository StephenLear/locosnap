## What Didn't Work (and Fixes)
- Same train (BR 406 at Utrecht) returned 4-5 different identifications across 5 scans — root cause: ICE 3 family has no disambiguation rules, cache miss on each different result. Fixed with Utrecht corridor rule in vision.ts.
- Account cross-contamination: two separate bugs, not one. SIGNED_OUT path bypassed clearHistory(), and AsyncStorage fallback returned previous user's data when new account had zero cloud spots. Both fixed.
- EU07 v1 opened on coaches not loco — recut to v2 using t=7-12 window (approach → full cab → close-up).

## Key Decisions
- **blueprint.tsx Image.getSize() deferred**: Sizing works because dimensions are known before render. Changing to onLoad would cause layout reflow. Not worth the risk until WatchdogTermination is confirmed to persist after animation fix.
- **Hold build until German tester feedback**: Better to consolidate all feedback into one build than ship incrementally. Apple submission gap: build 28 approved today, next submission Thursday/Friday minimum.
- **EU/UK/EU/UK content cadence**: Signals European positioning from day one, not just a UK app.
- **Backend push independent of app build**: Backend fixes deploy instantly via Render. No reason to hold them for the app build cycle.
- **RailUK response public, not defensive**: Naming the specific misidentifications by class, explaining the root cause, inviting ongoing feedback — turns public criticism into community ownership.

## Lessons Learned & Gotchas
- Always read docs/CHANGELOG.md before asserting what is or is not in a specific build. Do not rely on memory.
- Backend commits sitting unpushed are invisible to users — always end every backend commit with an explicit push-or-hold decision.
- After triggering an EAS build, monitor it and retrieve the APK link directly — never ask the user to supply it.
- The ICE 3 family (BR 403/406/407/408) looks visually identical from platform side angle. Location context (country, route) is the primary discriminator, not visual features.
- LU vintage stock (1938/1960/1972/A Stock) naming is date-based with very similar visual profiles — requires explicit per-class disambiguation rules in the vision prompt.
- AsyncStorage fallback in loadHistory() is a hidden data leak path: if new account has zero cloud spots, it falls through to local storage which may contain previous user's data.
- DR Class 156: only 4 were built — must be Legendary, not Uncommon. Verify rarity for any historically significant but visually unremarkable classes.
