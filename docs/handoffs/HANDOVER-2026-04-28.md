## Lessons Learned & Gotchas

- **Class-confusion vision bugs need statistical-default fixes, not just disambiguation rules.** Whack-a-mole on individual confusion pairs (BR 412 vs BR 408 → fixed → now BR 412 vs BR 403) is a losing pattern. The right pattern is "default to the statistically dominant class; positive evidence required for any other answer".
- **Anthropic prompt caching is content-keyed, so deploys auto-invalidate the cache.** No manual flush needed when shipping vision rule changes — the new prompt has a new cache key. 5-min TTL ephemeral cache rolls over fast.
- **Haiku 4.5 has a "refuse and apologise" failure mode** that lands meta-commentary like "I cannot provide reliable details" or "I must be honest" in the structured output fields when it's uncertain. Same regression class as BR 140 / BR 232 / BR 151. Hard prompt anchors with explicit anti-refusal clauses are the fix.
- **The `trainCache` in `services/trainCache.ts` is keyed by identified-class-name + language**, NOT by image hash. So a fresh scan of the same image always goes to vision again — no cache flush needed for "rescan to verify fix" workflows.
- **Render auto-deploy is 3-5 min from push.** When a tester rescan after deploy still shows the old behaviour, deploy completion is the first thing to verify (check `git log -1 --format='%h %ar' <commit>` against deploy expectation).
- **Cost spike on launch day is expected and not a bug.** Apr 28 ~$36 reflects real Play Store launch traffic + dev test cycles, not runaway. Tomorrow's baseline should settle to $5-15/day post-launch-shakedown.
- **Round-2 tester feedback is a value signal, not a complaint.** `pafawag.w.obiektywie` came back same-day with detailed structured corrections + reference source. Treat as evangelist (memory-logged + credit in commit message).
