## What Didn't Work (and Fixes)

- **settingsStore `setLanguage` not calling `i18n.changeLanguage`** — UI always rendered English regardless of stored preference. Root cause: store updated state and AsyncStorage but never told i18next. Fixed: both `setLanguage` and `initialize` now call `await i18n.changeLanguage(lang)`.
- **German locale detection test passing vacuously** — `jest.doMock("expo-localization", ...)` called after module registry already loaded the `"en"` mock. Fixed: `jest.resetModules()` must come before `jest.doMock` and the subsequent `require`. Order: `resetModules()` → `doMock(...)` → `require(module)`.
- **Double `i18n.changeLanguage` call in profile toggle** — `handleLanguageToggle` called `i18n.changeLanguage(next)` after `setLanguage(next)`, but the store already does this internally. Removed the redundant call from the handler.
- **`getTopTrains` key-split broken** — `getCacheKey` now produces `v7::en::class::operator`. Old split `[cls, operator] = key.split("::")` read version and language as class and operator. Fixed: `[, , cls, operator] = key.split("::")`.
- **`monitorBlueprintForCache` missing language** — German blueprints stored under English key, causing unnecessary regeneration. Fixed: `language` param added to function signature and forwarded at all call sites.
- **`scan.trialBanner_plural` not triggered** — i18next v3 pluralisation requires `count` in interpolation object. Original call only passed `remaining`. Fixed: `t("scan.trialBanner", { count: remaining, remaining })`.
- **String concatenation breaking i18n** — `{t("profile.level")} {levelInfo.index}` will break word order in non-English locales. Fixed: `t("profile.level", { number: levelInfo.index })` with translation string `"Level {{number}}"`.

## Key Decisions

- **EN/DE only for v1.0.8, architecture supports future languages** — `VALID_LANGUAGES` in route and `SUPPORTED_LANGUAGES` in store are arrays. Adding FR/PL/NL/CS requires: new locale file + 6 lines in i18n/index.ts + extending the valid languages arrays. No refactor needed.
- **First-launch gate, not settings-only** — Language picker shown before auth on every fresh install. Returning users skip it. Language can also be changed later via profile screen toggle.
- **Only narrative fields translated, not technical values** — Numbers, units, speed (250 km/h), operator names, class names stay in standard format. Only facts, rarity descriptions, and UI labels change language.
- **Vision/identification always in English** — The identification prompt is not language-sensitive. Only post-identification content (facts, rarity description, UI strings) changes language.
- **Cache per language** — A German scan of DB Class 101 and an English scan produce separate cache entries. Necessary because AI content is language-specific.
- **Hold EAS build until language picker is complete** — Language picker requires a new EAS build (1.0.8). The camera gallery toggle (committed in 1.0.7 era) is also waiting for this build.

## Lessons Learned & Gotchas

- **`jest.doMock` ordering is critical** — `jest.resetModules()` → `jest.doMock(...)` → `require(module)`. In the wrong order the module resolves against the previously registered mock and the doMock override is silently ignored. Test still passes but tests the wrong path.
- **Zustand store `initialize()` double-invoke in StrictMode** — React StrictMode (development) double-invokes useEffect. Without a guard, two concurrent `initialize()` calls both write to AsyncStorage. Guard: `if (!get().isLoading) return` at the top of `initialize()` since `isLoading` is set to `true` in the initial state.
- **i18next v3 pluralisation requires `count`** — The `_plural` suffix is only triggered when `count` is passed in the interpolation object. `remaining` alone does not trigger plural selection even if you also have `trialBanner_plural` defined.
- **Cache key format change = mandatory version bump** — Adding a new segment (language) to cache keys requires bumping the version so old entries are orphaned. Forgetting to bump means v6 English entries are served to German users.
- **`getTopTrains` key-split must match `getCacheKey` format** — Whenever `getCacheKey` format changes, `getTopTrains` destructuring must be updated to match the new segment positions. These two functions are tightly coupled and not type-checked against each other.
- **`expo-localization` must be in `app.json` plugins** — `app.json` was modified to add `"expo-localization"` to the `plugins` array. This is required for the EAS build to bundle the native module correctly. The modification was auto-committed in the final session commit.
- **Profile toggle handler should not call `i18n.changeLanguage` directly** — The store owns this side-effect via `setLanguage()`. Any component calling `setLanguage()` should not also call `i18n.changeLanguage()` — that's the store's responsibility. Calling both causes two overlapping language-change events.
