- **ffmpeg filter_complex concat with mixed inputs** — source was 1080x1920 h264 30fps; screen recording was 1206x2622 hevc 60fps. Single filter_complex with all three inputs failed with "Error reinitializing filters". Fix: encode each segment separately to identical specs, then concat with demuxer
- **Text position guessing without single-frame test** — text went through 4 rounds of correction (left-aligned → too small → cut off at top → correct). Fix now documented in video-editing skill: always test on single frame before full encode

## Key Decisions

- **Skip FCM entirely on Android (not just try/catch)**: JS try/catch cannot intercept native JNI crashes. Skipping the call entirely is the only reliable fix. Safe because push notifications have no backend implementation yet
- **Leave 3,200 kW error in EU07 video deliberately**: same comment-bait strategy that drove engagement on ET22 LEGENDARY rating. Polish rail fans will correct it
- **Crop top of source rather than mask badge**: after 6 failed drawbox attempts, crop is the correct tool. Loses ~350px of sky/arch top but the key loco shots are unaffected

## Lessons Learned & Gotchas

- **Badge pixel coordinates cannot be estimated from preview images** — scaled JPEG previews make coordinate measurement completely unreliable. For any future overlay removal, use `crop` after 2 failed drawbox attempts. The rule is now in the video-editing skill
- **ffmpeg filter_complex requires all inputs to have matching codec and fps** — always check source specs with `ffprobe` before attempting multi-source concat. If mismatched, encode each segment to a standard spec separately first
- **Always test text position on a single frame before full encode** — `ffmpeg -ss [t] -i input -vf "[filter]" -frames:v 1 test.jpg`. Saves multiple full encodes
- **Non-English email drafts must include English translation in the same response** — rule in memory and architecture doc, broken twice. The pre-output checklist in memory now forces the check before any output
- **Resend API key is always in ARCHITECTURE.md** — never ask the user for it. Checked and confirmed
