## Lessons Learned & Gotchas
- **Skill rule that bit us: "verify source frames before building" should extend to beat hero footage, not just screen recordings.** The existing rule targets screen-recording crops. The beat-1 Rheingold error happened because I verified one frame of Clip 3 (at 10s) and assumed the 1.5s window around that frame contained the same subject matter. Rule generalisation: for every beat, extract frames at both the in-timestamp AND the out-timestamp of the planned window and confirm the subject is present through the full duration. Logged under the BR 103 entry in CHANGELOG for future sessions to find.
- **`git status` on a handover shows modifications from the day's earlier session.** The Sr1/Sr2/Sr3 backend work from earlier on 2026-04-18 left `docs/ARCHITECTURE.md` with -2 lines uncommitted. Not this session's responsibility to resolve, but worth a `git diff` before any commits so those edits aren't lost or duplicated.
- **Build numbering convention.** The skill ARGUMENTS said "iOS build 42" — verified the last submitted build was 41 and autoIncrement would hand out 42 for v1.0.20. If user says build N in future sessions, check line 392 of `docs/ARCHITECTURE.md` for the latest live build to confirm the number makes sense.
- **`stats 18:4` folder naming.** User uses `stats DD/M` folder names on Desktop (e.g. "stats 18:4" = 18 April). Colon is safe in macOS filenames via Finder but shows as `/` there. Glob with the name verbatim works.
