## What Worked
- `key={i18n.language}` on `<Tabs>` is the correct React Navigation pattern for forcing tab label re-evaluation on locale change — minimal, no state required, no extra hooks
- Splitting `git add` and `git commit` into separate Bash tool calls when file paths contain parentheses (avoids shell subshell-parsing errors)

## What Didn't Work (and Fixes)
- `git add "frontend/app/(tabs)/_layout.tsx" && git commit ...` in a single Bash call — bash treated `(tabs)` as a subshell when chained with `&&`. Fixed by issuing `git add` as a separate Bash call first, then `git commit` separately.

## Key Decisions
- **`key` prop on `<Tabs>` not a full remount of tab screens**: The `key` remounts the navigator shell (forcing screenOptions re-read) but tab screen components themselves are not fully destroyed — scroll positions and local state are preserved. Acceptable trade-off for language switching.

## Lessons Learned & Gotchas
- Shell special characters (`(`, `)`, `[`, `]`) in file paths cause parsing errors when combined with `&&` in a single Bash call. Always split `git add` and `git commit` into separate tool calls when paths contain these characters.
- Completing pre-build work (fixes, tests, commit) does NOT constitute permission to trigger a build. Always ask first.
- React Navigation caches `screenOptions` at mount — any `t()` calls inside `screenOptions` will not re-run when i18n language changes unless the navigator is remounted via `key`.
