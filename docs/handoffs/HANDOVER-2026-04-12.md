### Frontend -- card-reveal.tsx hooks crash fix (submitted for App Store review)

Sentry REACT-NATIVE-C: "Rendered fewer hooks than expected" crash on CardRevealScreen. 1 event, 1 user, iPhone 14 iOS 26.3.1, release 1.0.17 (38).

Root cause: Four `useMemo` hooks (frontInterpolate, backInterpolate, frontOpacity, backOpacity) were declared after the early return guard (`if (!currentTrain || !currentRarity) { return ... }`). When the guard fired, React counted fewer hooks than expected.

Fix: Moved all four useMemo calls above the early return. Added inline comment explaining the constraint. Code-reviewed by coderabbit agent -- passed all five checks (fix correct, no remaining hooks below return, useMemo safe with null state, no other early returns, no other crash-risk patterns).

v1.0.18 build 40 submitted to App Store review. Release notes provided in EN and DE.

## Lessons Learned & Gotchas

- **Always bump the version before an iOS build if the current version is already live** -- EAS auto-increments buildNumber but NOT CFBundleShortVersionString. Apple rejects any build with a version that's already been approved. This has happened before and a skill rule has now been added to prevent it.
- **UK downloads are 2:1 over Germany** -- this is a critical data point that was not in any previous handover. The German content strategy has been optimised for German view share, but downloads are what matter. If UK converts better per view, the content strategy may need rebalancing.
- **The pattern interrupt hook ("2026 IST SCHLUSS" / "SEIT 1968 BALD WEG") produces better engagement quality even if TikTok reports early drop-off** -- TikTok said "most viewers stopped at 0:01" on BR 101 DE, but the actual avg watch (5.57s on a 10s video) and full-watch (26%) are the best numbers to date. The "stopped at X" metric may be measuring scroll-past impressions, not engaged viewers.
- **DSB Class ER numbering: 2xxx = S-tog EMU, 15xx = Class ME diesel** -- fleet number alone is definitive. The vision model had no DSB-specific rules before today.
- **RevenueCat App Store connectivity errors (REACT-NATIVE-D) are transient Apple issues** -- handled errors, no app-side fix needed. Archive on sight.
