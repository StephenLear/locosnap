### 1.0.2 — What Was In It (Crash Root Cause Analysis)
Changes shipped in 1.0.2:
- **HEIC → JPEG conversion** — iOS gallery picker now converts HEIC images via `expo-image-manipulator` before upload (was rejecting with "invalid file type")
- **Vision prompt fixes** — Desiro family naming ("Desiro Classic/UK/City"), DB German class disambiguation (BR 642/643/644)
- **trainSpecs.ts** — Explicit km/h rule for European operators (was showing mph for German trains)
- **rarity.ts** — German fleet awareness rules (DB Class 643/644, 628 now classified as rare/withdrawn correctly)
- **Supabase auth** — Magic link changed to 8-digit OTP code, 24hr expiry (was 1hr)
- **Supabase SMTP** — Custom Resend SMTP configured for email sending

**Most likely crash causes in 1.0.2:**
1. **Backend cold-start timeout** — Render.com free/starter tier spins down after inactivity. As new users (from +200% download spike) hit the app for the first time, the backend was cold → identify request times out → app crashes. **This is the prime suspect.** 1.0.3 added a pre-warm health check on app launch specifically to prevent this.
2. **ImageManipulator edge case** — HEIC conversion could fail on certain iOS versions or corrupted images, potentially causing an unhandled rejection crash.

**Verdict:** 1.0.3 pre-warm fix almost certainly addresses the dominant crash cause. Watch Monday/Tuesday's App Store Connect figures — if crashes drop sharply as 1.0.3 adoption grows, this is confirmed resolved. If not, ImageManipulator is next suspect.

## Key Decisions
- **5 scans/day → 10 scans/month**: Monthly model gives casual users enough headroom while power users hit ceiling and upgrade; daily reset too easy to game
- **Guest limit = 3 total**: Enough to experience the value, forces sign-up before heavy use
- **Crashes are 1.0.2, not 1.0.3**: No emergency hotfix needed — watch Monday's figures; 1.0.3 pre-warm fix may already resolve the main cause
- **Cache needs Redis, not file**: Render.com ephemeral FS means file cache is worthless in production across deploys

---

## Lessons Learned & Gotchas
- App Store Connect crash data is **24-48hrs behind** — always check version release dates before panicking
- Guest users in `canScan()` bypass all limits — any limit change must also apply to guests
- `MAX_HISTORY = 50` (history list size) and `MAX_DAILY_SCANS = 5` (scan rate limit) are completely separate — Czech tester hit history cap, not scan cap
- Render.com ephemeral FS: any file written at runtime (cache, logs, temp) is gone on next deploy — use Redis/external storage for anything that must persist
- Dual-voltage Czech trains (ČD 362, etc.): AI prompt lists "Dual-voltage Electric" as an option but doesn't enforce it — needs explicit Czech/Slovak voltage rules added to SPECS_PROMPT
- ICE 1 weight: AI returned 1 ton — needs validation: `if (weight < 10) reject` before display
- Testers appearing on leaderboard requires: account created + signed in + scan completed + Supabase sync succeeded
