## Lessons Learned & Gotchas

- **Sentry "Es kracht" / German idiom literal-vs-figurative**: when a tester comment has a plausible literal-failure interpretation alongside a colloquial-positive interpretation, default to the literal if ANY relevant Sentry alert is open. The figurative read forced a recovery exchange.
- **Chrome MCP + auth-walled SaaS**: Sentry, Apple App Store Connect, Google Play Console, and similar tools are accessible via Claude in Chrome MCP using the existing user session. WebFetch fails on these. Use Chrome MCP whenever auth-walled SaaS data is needed.
- **yt-dlp for TikTok metadata**: `yt-dlp --skip-download --print "...%(field)s..." <url>` reliably extracts title, description, upload_date, view_count, like_count, comment_count for any public TikTok video. Permanent fallback when `tiktok_stats.md` doesn't have history.
- **expo-image-picker 17.x has no fix for generic Activity recreation**: 17.0.10 → 17.0.11 diff is only a `null` mime type fix. Don't waste a bump expecting the lifecycle fix.
- **The Activity-recreation lifecycle bug is upstream, not user-fixable** at app code level. v1.0.24 mitigations (auto-retry + camera fallback) are best-effort recovery, not root-cause fix. File upstream issue with Expo for visibility.
- **TikTok per-video logging discipline started 2026-04-29.** Pre-2026-04-29 ads need yt-dlp recovery. Going forward: log every ad to `tiktok_stats.md` at posting time, not just the high performers.
- **Don't auto-correct tester corrections that are deliberate ad strategy** — Eiffel comparison on BR 412 ICE 4 was a scroll-stop hook, not a fact claim. User direction: trust their domain expertise on ad style.
- **Skowarcz clip has ET22 AND Newag Dragon at DIFFERENT timestamps**, not in the same frame. Old-vs-new visual contrast assumption was wrong. Frame-check before relying on described visual content.
- **Polish ASS subtitles MUST be written via Write tool** — heredoc corrupts `Ł` and other diacritics. Memory rule already documented; held this session.
- **Migration 012 paste error**: copy from a markdown code-fenced block in chat included the leading "S" from "```sql", resulting in `SCREATE TABLE` syntax error. Recovery: paste the SQL again as plain text (no fences). For future migrations: present SQL as plain text in chat or instruct the user to copy from the migration file directly.
- **EAS Submit's `releaseStatus: draft`** means the AAB lands in Play Console as a draft release entry — but the user navigating to "Create new release" creates a SEPARATE empty release rather than finding the draft. Walkthrough must explicitly guide to the existing draft via the Production track overview, NOT the "Create new release" button.
- **Apple/Google review unpredictability**: Google approved v1.0.23 within hours; Apple still in review at session close. Google can be faster than Apple for Android-specific fixes. Don't assume parallel approval timing.
- **Internal testing track stale-versionCode hazard**: even with no active testers, stale low versionCode in Internal track can trigger Play quick-check false positives during production review. Either pause the track or promote the production AAB into it.
