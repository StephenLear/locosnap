## Lessons Learned & Gotchas

- **Training-data edge cases exist for rare liveries.** Heritage livery repaints on niche trains (Swallow on 91127) have too few training examples for the vision model to reliably identify even with a prompt rule. Standard liveries work fine. Don't try to fix these via prompt engineering — accept the edge case and use a more common photo for demos.
- **Publishing a "NUR NOCH X" scarcity claim without a region qualifier is risky.** German rail enthusiasts know which Verwaltungsbereich a train runs in. A specific number without the qualifier invites factual challenges. Either include the qualifier in the video text or be ready to clarify in comments.
- **The `dailyscansresetat` column in Supabase is dead code now** — the backend doesn't read or write it since the 2026-04-14 flip. It remains in the schema but can be dropped in a future cleanup session.
- **Resend Cloudflare User-Agent header is critical** — any Python-urllib script calling the Resend API must include `User-Agent: LocoSnap-TesterMailer/1.0 (curl-equivalent)` and `Accept: application/json` headers or it gets 403/error 1010. This is documented in ARCHITECTURE.md Section 10.
- **ETCS as a scroll-stopper hook works** — "KILLED BY ETCS" creates immediate curiosity (what's ETCS?) that forces viewers to stay for the reveal. More effective than generic scarcity ("ONLY 12 LEFT") as the opening line.
- **3 new testers appeared on Google Play without being recruited** — `jakubek.rolnik`, `mf.bruch`, `stevelear51` were already on the Play Console closed testing list but not in the memory file. The memory file was stale. Always cross-check against the Play Console source of truth when distributing updates.
- **Trainspecs.ts hardcoded overrides are essential after a vision rule addition** — every new vision rule should have a matching trainSpecs entry to prevent Wikidata/AI hallucination on the specs once the vision layer correctly identifies the class.
