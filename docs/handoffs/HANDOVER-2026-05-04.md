## What Worked

- **Cherry-picking the v1.0.25 hotfix scope from the larger leaderboard plan** — kept tester YXNSST unblocked without forcing all of Phase 2 to be ready first. Same branch can carry both, just two stages of commits.
- **`eas.json` `appVersionSource: remote` + `autoIncrement: true`** — never had a versionCode collision, EAS managed it cleanly.
- **Render notes draft alongside DE typography** — typing `„X"` (German low-9 + ASCII straight quote) broke JSON parsing on `de.json`. Fixed by switching to proper `„X"` (U+201E + U+201C) typographic pair. Worth flagging next time DE quotes appear.
- **TDD on the verification classifier** — writing failing tests for the new PERSONAL/UNVERIFIED split before changing the implementation surfaced 5 places where the existing test expectations needed updating to match the new semantic. Saved a round of "tests pass but they're testing the wrong thing".
- **Server-side XP computation via SECURITY DEFINER RPC** — `award_weekly_xp_for_spot` reads spot fields directly so the client can't forge XP values. Cleaner than a backend endpoint (no extra HTTP round-trip) and securer than a client-side computation.
- **ScheduleWakeup for build polling** — set 15-min wakeup, came back exactly when Android was done. iOS needed one more 10-min poll. No wasted cache cycles.

## Lessons Learned & Gotchas

- **Android `insets.bottom = 0` on non-edge-to-edge**. The `Math.max(insets.bottom, 16)` pattern from `blueprint.tsx` is fine for iOS but inadequate for Android 3-button nav. Use `Math.max(insets.bottom, spacing.xl + spacing.md)` (36px floor) or larger when the screen has a primary tap target near the bottom edge.
- **Don't submit one platform of a multi-platform release without confirming timing**. Even though `releaseStatus: draft` means no review clock starts on submit, the user may want to coordinate. Always ask first.
- **DE typographic quotes break JSON if you use ASCII closing quote**. `„X"` is broken. `„X"` is correct (U+201E opening + U+201C closing).
- **Per-class diminishing-returns needs `class_key` on the audit table**. Don't try to JOIN through `spots` — denormalize.
- **`appVersionSource: remote` + `autoIncrement: true` in `eas.json`** = EAS auto-manages versionCode + buildNumber. Just bump `version` in `app.json` and EAS handles the rest. Verify with `eas build:version:get` if you're worried.
- **Migration plan and design doc terminology can drift**. Always reconcile to codebase before extending. We had VERIFIED/PERSONAL/UNVERIFIED in the design doc but `verified-live` / `verified-recent-gallery` / `unverified` in the code. Migration 013 came in with the 3 codebase values; we extended to 4 today.
- **Server-side computation > client-side computation for any value that affects ranking/competition**. Even with auth gates, anything the client supplies as a parameter can be forged. `award_weekly_xp_for_spot` reads from `spots` directly.
- **TikTok comment replies are 150 chars max — DMs have higher limits**. Often forces choosing 1-2 of 3 points to address. Address the strongest substantive technical point and let the audience inferences fill the rest.
- **Set explicit timing expectations in tester replies**. "morgen oder Dienstag im Play Store" beat vague "next week". Reduces follow-up "where is it?" messages.
