## Lessons Learned & Gotchas

- **Facts-layer hallucinations are routinely worse than specs-layer hallucinations on the same class.** Specs are constrained by typed fields and KNOWN_SPECS overrides. The facts narrative paragraph is free-form prose, so the LLM pads short fact lists with plausible fabrications (cross-border operation, sub-variant numbering, modular configurations). On a high-coverage class (BR 101 / 151 / 110) the override block keeps the prose accurate. On a low-coverage class (ÖBB 4020) the LLM invents entire deployment histories. **Future correction triage:** always pull the FULL scan card (front + back + Fun Facts + Notable Events) before scoping a fix.
- **Backup drives drop offline mid-session.** USB sleep behaviour. The "Steve backup" drive did this once today between inventory and staging. Replug and resume. Captured here so the next session doesn't waste a diagnostic round on "drive missing".
- **Scan-recording sources predate fixes.** Several screen recordings on the backup drive (SU46 May 21, BR 151 April 23) were captured BEFORE the same-session backend fixes shipped. The recordings still contain the wrong-fact frames even though the live backend serves correct facts now. **Future ad builds:** if you use an old screen recording, audit it against the current backend output before locking the cut, or re-record on a fresh scan.
- **Untracked creative assets (`ad_assets/endcards/endcard_pl.mp4`) survived from an earlier session and were not committed.** Memory file `ad_assets_endcards.md` lists this folder as the canonical endcard location, but the contents have never been committed to git. If the local disk loses these files (as happened earlier this session with the BR101/151/120 source clips), they have to be rebuilt from scratch. Recommendation: commit the canonical endcards to the repo (DE/PL/EN, ~30KB each) so they're reliably available. Decision deferred to user.
- **The `--no-edit` flag is not valid for `git rebase`** (per CLAUDE.md). Did not need it this session, but noting because the rule lives in the doc and is easy to slip back into.
- **Heredoc commit messages fail on apostrophes / umlauts.** Used file-based `git commit -F /tmp/commit_msg.txt` for all four commits this session per CLAUDE.md. All succeeded first try.
