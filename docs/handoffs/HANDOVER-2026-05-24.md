## Lessons Learned & Gotchas

- **Facts-layer hallucinations are routinely worse than specs-layer hallucinations on the same class.** Specs are constrained by typed fields and KNOWN_SPECS overrides; the facts narrative paragraph is free-form prose, so the LLM pads short fact lists with plausible fabrications. Now confirmed as a **class-wide pattern, not class-specific** — two unrelated classes in 24 hours (ÖBB 4020, VR Sr1) failed the same way. Every class with a KNOWN_SPECS entry but no `trainFacts.ts` bullet is at risk. A one-time audit of KNOWN_SPECS keys vs `trainFacts.ts` bullet list would surface every at-risk class — flagged for the next quiet-content session.
- **Word-for-word matches between bug reports and historical wrong output = cache hit, not fresh hallucination.** Five seconds of careful reading vs an hour of prompt-vs-cache hypothesis testing.
- **Every `KNOWN_SPECS` lookup key needs a matching `CLASS_INVALIDATIONS` entry** for any class that gets corrected. Codified in `backend_cache_invalidation_pattern.md` with a 4-step checklist. First proactive application (VR Sr1) needed no retrofit — pattern is now validated.
- **`/api/health` does NOT carry a build-version field.** `version: "1.0.0"` is hardcoded and doesn't change between deploys. Use `cache.totalEntries` (resets to 0 on restart) as the "new build live" signal. Adding a `buildSha` field sourced from a `GIT_SHA` env var at build time would close this gap cheaply.
- **Translation rule must persist across every iteration of a draft.** Codified in the memory update. Specific failure modes documented: name-substitution passes, wording tweaks, "final version" presentations, subject-line presentations. Pre-send check: "is there a complete English translation of the latest version in this same message?" If no, the message is incomplete.
- **Resend returns 200+ID on suppressed addresses.** API-acceptance ≠ delivered. After ANY non-trivial Resend send, pull the dashboard and confirm `last_event = delivered` before declaring delivery successful.
- **The "Pro grant" reflex must be checked against `policy_pro_grants_discontinued.md` every time** a paying customer's subscription is at risk. The discipline is to find an actual fix in the product / subscription flow rather than papering over with a comp.
- **Backup drives drop offline mid-session.** USB sleep behaviour. Replug and resume.
- **Scan-recording sources predate fixes.** Old screen recordings may contain wrong-fact frames even though the live backend now serves correct facts. Before locking any cut that uses an old recording, audit against current backend output or re-record on a fresh scan.
- **Heredoc commit messages fail on apostrophes / umlauts.** Used `git commit -F /tmp/commit_msg.txt` for every commit this session per CLAUDE.md. All succeeded first try.
